## 开篇介绍

云原生（Cloud Native）是一种构建和运行应用的 **方法论**，充分利用云计算的优势，构建弹性、可扩展、易于管理的应用。云原生技术包括容器化、微服务架构、服务网格、声明式 API、DevOps 等。

不得不说，这个词听起来就挺有逼格。

**为什么要学云原生？**

云原生是当前企业级应用开发的主流方向，几乎所有互联网公司和传统企业都在向云原生转型。云原生技术可以提高应用的可靠性、可扩展性、可维护性，降低运维成本。

目前云原生工程师的薪资非常高，一线城市的云原生架构师平均薪资在 40-60K，资深专家可以达到 60-150K+。

![](https://pic.yupi.icu/1/image-20251203121708577.png)

云原生的核心技术包括容器化（Docker）、容器编排（Kubernetes）、微服务架构、服务网格等。Kubernetes 是云原生的核心技术，是容器编排的事实标准。

在 AI 时代，云原生的弹性伸缩、资源调度等特性，非常适合 AI 模型的训练和推理。

## 整体学习建议

1）容器化是云原生的基础，一定要先学好 Docker。理解容器的概念、镜像的构建、容器的运行等。

2）Kubernetes 是云原生的核心技术，要重点学习。K8s 的概念很多，学习难度较大，要有耐心，边学边实践。

4）CNCF 有大量优秀的开源项目，比如 Prometheus（监控）、Envoy（服务网格）、Helm（包管理）等。如果你的目标是大厂或者高级工程师，建议多关注 CNCF 生态。

## 阶段 1：容器化基础（Docker）（15-25 天，仅供参考）

容器化是云原生的第一步，要深入理解容器的概念和优势。容器相比虚拟机更轻量、启动更快、资源利用率更高。

详细的 Docker 学习路线可以参考：[Docker 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754953097949186)



### 学习目标

掌握 Docker 容器技术，能够容器化应用。



### 知识点

**Docker 基础【必学】：**

- 容器和虚拟机的区别
- Docker 的安装和使用
- 镜像（Image）和容器（Container）
- Docker 命令（run、pull、push、build 等）

**Dockerfile【必学】：**

- Dockerfile 的编写
- 常用指令（FROM、RUN、COPY、CMD、ENTRYPOINT 等）
- 镜像的分层结构
- 多阶段构建

**Docker Compose【必学】：**

- Docker Compose 的使用
- docker-compose.yml 配置
- 多容器应用

**镜像仓库【建议学】：**

- Docker Hub
- 阿里云镜像仓库
- Harbor 私有仓库【可不学】



### 学习建议

1）Dockerfile 是构建镜像的脚本，要掌握 Dockerfile 的编写技巧。多阶段构建可以减小镜像体积，提高安全性。

2）Docker Compose 可以定义和运行多容器应用，适合本地开发和测试。在生产环境中，一般使用 Kubernetes。



### 学习资源

- ⭐ [Docker 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754953097949186)：完整的 Docker 学习路线
- [Docker 官方文档](https://docs.docker.com/)：官方文档



## 阶段 2：Kubernetes 容器编排（20-40 天，仅供参考）

Kubernetes 是云原生的核心技术，概念很多，学习曲线较陡。建议先掌握核心概念（Pod、Service、Deployment），再逐步学习高级特性。

详细的 Kubernetes 学习路线可以参考：[Kubernetes 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754993086443521)



### 学习目标

掌握 Kubernetes，能够部署和管理容器化应用。



### 知识点

**Kubernetes 基础【必学】：**

- Kubernetes 的架构
- 核心概念（Pod、Service、Deployment、Namespace）
- kubectl 命令
- YAML 配置文件

**工作负载【必学】：**

- Deployment（无状态应用）
- StatefulSet（有状态应用）
- DaemonSet
- Job 和 CronJob

**服务发现和负载均衡【必学】：**

- Service（ClusterIP、NodePort、LoadBalancer）
- Ingress
- Ingress Controller

**配置和存储【必学】：**

- ConfigMap 和 Secret
- Volume 和 PersistentVolume
- StorageClass

**Helm【建议学】：**

- Helm 的使用
- Chart 的编写



### 学习建议

1）Pod 是 K8s 的最小调度单元，一般包含一个或多个容器。Deployment 用于管理 Pod，提供滚动更新、回滚等功能。

2）Service 提供服务发现和负载均衡，让 Pod 之间可以相互访问。Ingress 提供 HTTP 路由，让外部可以访问集群内的服务。



### 学习资源

- ⭐ [Kubernetes 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754993086443521)：完整的 K8s 学习路线
- [云原生 Kubernetes 教程](https://www.bilibili.com/video/BV13Q4y1C7hS)：K8s + Docker + KubeSphere + DevOps（实战时要注意可能有些配置发生了调整）
- [云原生实战](https://kubesphere.io/zh/learn/)：KubeSphere 教程



## 阶段 3：微服务架构（15-25 天，仅供参考）


微服务架构是将应用拆分为多个独立服务的设计模式，配合容器和编排工具实现灵活的云原生应用。



### 学习目标

云原生应用一般采用微服务架构，要理解微服务的设计原则和最佳实践，掌握微服务开发和部署。



### 知识点

**微服务基础【必学】：**

- 微服务架构的特点
- 服务拆分和设计
- 微服务的通信（REST、gRPC）

**Spring Cloud / Go Micro【建议学】：**

- 服务注册和发现
- 服务网关
- 配置中心
- 熔断降级

**云原生应用开发【必学】：**

- 12 因素应用
- 无状态设计
- 配置外部化
- 健康检查

### 学习建议

1）12 因素应用是云原生应用的设计准则，包括代码库、依赖管理、配置、后端服务等 12 个方面。要理解这些准则的含义。

2）云原生应用要设计为 **无状态** 的，状态存储在外部（如数据库、缓存）。这样可以方便地水平扩展和故障恢复。



### 学习资源

- [Spring Cloud 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754562939596801)：完整的微服务学习路线
- [12 因素应用](https://12factor.net/zh_cn/)：云原生应用设计准则



## 阶段 4：服务网格 Service Mesh（10-20 天，仅供参考）

Service Mesh 是云原生的高级特性，用于管理微服务之间的通信。Service Mesh 在应用无感知的情况下提供流量管理、安全、监控等功能。

![](https://pic.yupi.icu/1/https%253A%252F%252Fsubstack-post-media.s3.amazonaws.com%252Fpublic%252Fimages%252F71543b63-9ac4-4c4f-bea9-103cd25fcdc3_1992x1514.png)



### 学习目标

理解服务网格的概念，掌握 Istio 的基本使用。



### 知识点

**Service Mesh 基础【必学】：**

- Service Mesh 的概念
- 数据平面和控制平面
- Sidecar 模式

**Istio【建议学】：**

- Istio 的架构
- 流量管理
- 安全（mTLS）
- 可观测性

**其他 Service Mesh【可不学】：**

- Linkerd
- Consul Connect

### 学习建议

1）Istio 是最流行的 Service Mesh 实现，基于 Envoy 代理。Istio 的学习曲线较陡，建议先掌握 Kubernetes，再学习 Istio。

![](https://pic.yupi.icu/1/image-20251203122355697.png)

2）Service Mesh 相对复杂，了解即可。实际项目中不一定需要 Service Mesh，要根据业务规模选择。



### 学习资源

- [Istio 官方文档](https://istio.io/latest/zh/docs/)：官方文档



## 阶段 5：可观测性（10-20 天，仅供参考）

可观测性是云原生的重要组成部分，包括监控（Metrics）、日志（Logging）、追踪（Tracing）三大支柱。



### 学习目标

掌握云原生应用的监控、日志、追踪技术。



### 知识点

**监控【必学】：**

- Prometheus（指标采集）
- Grafana（可视化）
- 告警（AlertManager）

**日志【必学】：**

- EFK（Elasticsearch + Fluentd + Kibana）
- 日志采集和分析

**分布式追踪【建议学】：**

- Jaeger
- 链路追踪



### 学习建议

1）Prometheus 是云原生监控的标准，Grafana 是可视化的首选。要掌握 Prometheus 的使用和 PromQL 查询语言。

建议大家观看鱼皮的 [Prometheus + Grafana + ARMS 大厂级别监控视频教程](https://www.bilibili.com/video/BV1QPYDztEtW/)，很快就能学会如何使用了。

2）分布式追踪可以追踪请求在微服务之间的调用链路，帮助排查性能问题。



### 学习资源

- [Prometheus 官方文档](https://prometheus.io/docs/introduction/overview/)：官方文档
- [Grafana 官方文档](https://grafana.com/docs/)：可视化工具
