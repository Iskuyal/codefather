## 开篇介绍

性能测试是软件测试中的重要分支，专注于评估和优化系统的性能表现。

在互联网时代，用户对系统性能的要求越来越高，页面加载慢几秒就会流失大量用户。从电商大促到春运抢票，从直播高峰到游戏开服，高并发场景无处不在。性能测试工程师的职责就是在系统上线前发现性能瓶颈，通过压力测试验证系统能力，为系统优化提供数据支持，保证系统在高负载下依然稳定运行。

具体来说，性能测试是通过模拟真实用户的操作行为和访问模式，对系统进行压力测试，评估系统的响应时间、吞吐量、并发能力、资源使用率等性能指标。性能测试不仅仅是执行测试，还包括性能需求分析、测试方案设计、瓶颈分析、性能调优等完整流程。它需要测试、开发、运维等多方协作，是一项综合性很强的工作。

**为什么要学性能测试？**

首先，性能测试人才稀缺且薪资高。性能测试相比功能测试门槛更高，需要更深的技术功底，因此优秀的性能测试工程师非常稀缺。根据 2025 年的市场调研，有经验的性能测试工程师年薪可达 25-60 万。

![](https://pic.yupi.icu/1/image-20251128163220945.png)

即使你不走测试方向，鱼皮也建议你学习下性能测试的知识，因为性能测试需要理解系统架构、数据库、网络、操作系统等多方面知识，能够培养全栈视野。

鱼皮个人认为，性能测试是一个很能带来成就感的工作。想一想，通过你的测试和优化，成功将系统性能提升 10 倍，这种成就感是其他工作很难比的。

性能测试的核心价值在于保证系统在高负载下的稳定性和可用性。通过性能测试，可以提前发现性能瓶颈，避免线上故障，提升用户体验，降低运营成本。性能测试广泛应用于各种场景，包括电商大促、春运抢票、游戏开服、直播高峰、新系统上线、重大版本发布等。



### 就业方向

学习性能测试可以助你从事多个方向的工作：

1. 性能测试工程师：负责性能测试的设计和执行，包括压力测试、负载测试、稳定性测试等工作。
2. 性能优化工程师：不仅进行性能测试，还负责性能瓶颈分析和系统调优工作。
3. 测试开发工程师：开发性能测试工具和平台，提升性能测试效率。关于测试开发的详细学习，可以查看 [测试开发学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755654066810882)
4. DevOps 工程师：将性能测试集成到 CI/CD 流程，实现持续性能测试。
5. 架构师/技术专家：性能测试经验可以帮助你更好地设计高性能系统。

## 阶段 1：性能测试基础（5-20 天，仅供参考）

### 学习目标

了解性能测试的基本概念和方法，掌握性能测试的流程和指标。

### 知识点

**性能测试概念【必学】：**

- 什么是性能测试
- 性能测试的目的
- 性能测试 vs 功能测试
- 性能测试的分类
  - 基准测试
  - 负载测试
  - 压力测试
  - 稳定性测试
  - 峰值测试

**性能指标【必学】：**

- 响应时间（Response Time）
- 吞吐量（TPS/QPS）
- 并发用户数
- 错误率
- 资源使用率（CPU、内存、磁盘、网络）

**性能测试流程【必学】：**

- 需求分析
- 测试计划
- 脚本开发
- 测试执行
- 结果分析
- 调优建议
- 报告编写

**后端基础知识【必学】：**

- HTTP 协议
- 数据库基础
- Linux 基础
- 网络基础



### 学习建议

1）性能测试需要一定的技术基础，不要完全零基础就学习性能测试。建议先了解 HTTP 协议、数据库、Linux 等基础知识，对系统架构有基本认识。

2）要理解不同类型性能测试的区别。基准测试是测试单用户性能，负载测试是测试多用户性能，压力测试是测试系统极限，稳定性测试是测试长时间运行。不同的测试目的不同，设计方案也不同。

3）性能指标要准确理解。响应时间是用户最关心的指标，吞吐量反映系统处理能力，并发用户数不等于在线用户数。要理解这些指标的含义和计算方法。

4）性能测试不是简单的工具使用，更重要的是测试思路和分析能力。要学会分析性能瓶颈，定位问题根源，提出优化建议。

## 阶段 2：JMeter 工具（5-30 天，仅供参考）


JMeter 是最流行的开源性能测试工具，功能强大且完全免费。JMeter 基于 Java 开发，跨平台，支持多种协议（HTTP、JDBC、FTP、JMS 等）。必须熟练掌握 JMeter。



### 学习目标

掌握 JMeter 性能测试工具，能够使用 JMeter 进行性能测试。

![](https://pic.yupi.icu/1/d365bcapiloadtestjmeter_15-20251128163514026.jpg)



### 知识点

**JMeter 基础【必学】：**

- JMeter 架构
- 测试计划（Test Plan）
- 线程组（Thread Group）
- 取样器（Sampler）
- 监听器（Listener）
- 断言（Assertion）

**HTTP 请求测试【必学】：**

- HTTP 请求配置
- 请求头和参数
- Cookie 管理
- Session 管理
- 文件上传下载

**参数化【必学】：**

- CSV Data Set Config
- 用户参数
- 变量和函数
- 参数关联

**JMeter 高级特性【必学】：**

- 逻辑控制器
- 定时器
- 前置处理器和后置处理器
- 正则表达式提取器
- JSON 提取器

**分布式测试【必学】：**

- 主从架构
- 远程测试
- 结果聚合

**结果分析【必学】：**

- 聚合报告
- 图形结果
- TPS、响应时间分析
- 错误分析



### 学习建议


1）线程组是 JMeter 的核心组件，用于模拟并发用户。要理解线程数、Ramp-Up 时间、循环次数的含义。合理设置这些参数，才能模拟真实的用户访问模式。

2）参数化是 JMeter 的重要功能，可以让测试脚本使用不同的数据。比如登录测试，需要多个用户账号；搜索测试，需要多个搜索关键词。要掌握 CSV Data Set Config 的使用，实现数据驱动测试。

3）结果分析是性能测试的关键环节。要学会查看聚合报告，分析 TPS、响应时间、错误率等指标。根据测试结果，判断系统是否满足性能要求，是否存在性能瓶颈。

4）建议从简单的场景开始练习，比如测试一个登录接口、一个查询接口。逐步增加难度，测试复杂的业务流程、混合场景等。在实践中理解 JMeter 的各种功能。

5）强烈推荐通过鱼皮的实战项目学习 JMeter。鱼皮的 [聚合搜索平台](https://www.codefather.cn/course/1790979621621641217) 和 [代码生成器共享平台](https://www.codefather.cn/course/1790980795074654209) 都从 0 到 1 带你入门并实战 JMeter 压力测试，包括线程组配置、HTTP 请求设置、结果分析等，比单纯学习理论效果好得多。

## 阶段 3：性能调优（10-25 天，仅供参考）


性能调优是性能测试工程师的高级技能。仅仅发现性能问题还不够，还要能够分析问题原因，提出优化方案。这需要对系统有全面的理解，包括应用、数据库、服务器、网络等各个层面。



### 学习目标

掌握性能调优方法，能够分析性能瓶颈并提出优化方案。

### 知识点

**性能瓶颈分析【必学】：**

- CPU 瓶颈
- 内存瓶颈
- 磁盘 I/O 瓶颈
- 网络瓶颈
- 数据库瓶颈
- 应用代码瓶颈

**数据库优化【必学】：**

- SQL 优化
- 索引优化
- 连接池配置
- 慢查询分析

**应用优化【必学】：**

- 代码优化
- 算法优化
- 缓存使用
- 异步处理
- 批量处理

**服务器优化【建议学】：**

- JVM 调优
- Tomcat 调优
- Nginx 配置优化
- 操作系统参数调优

**监控工具【必学】：**

- Linux 性能监控（top、vmstat、iostat）
- JVM 监控（JVisualVM、JProfiler）
- 数据库监控
- APM 工具（Application Performance Monitoring）



### 学习建议


1）数据库往往是性能瓶颈的主要来源。要学会分析慢查询日志，使用 EXPLAIN 分析执行计划，优化 SQL 和索引。数据库优化的投入产出比一般很高，优化一条 SQL 可能提升整体性能 10 倍。

![](https://pic.yupi.icu/1/1763393924660-6d74991d-1858-4aba-adb0-cc5615450e3a.png)

2）缓存是提升性能的重要手段。合理使用缓存可以大幅减少数据库访问，提升响应速度。要理解缓存的使用场景和注意事项，避免缓存穿透、缓存击穿等问题。

3）监控工具是性能调优的重要辅助。要学会使用各种监控工具，实时查看系统的资源使用情况。通过监控数据，可以快速定位性能瓶颈。

4）**性能优化没有银弹，需要根据实际情况采取不同的措施。**

建议从成本最低、效果最明显的优化开始，比如添加索引、启用缓存、优化 SQL 等。不要一开始就进行复杂的优化，如分库分表、微服务拆分等。

## 阶段 4：高级性能测试（20-30 天，仅供参考）

### 学习目标

掌握高级性能测试技术，包括 Gatling、k6 等现代化工具的使用。



### 知识点

**Gatling【建议学】：**

- Gatling 架构
- Scala DSL 脚本
- 场景设计
- 结果分析
- 和 JMeter 对比

**k6【建议学】：**

- k6 架构
- JavaScript 脚本
- 云端测试
- 性能监控集成

**性能测试平台【建议学】：**

- 云压测平台
- 自建性能测试平台
- 平台对比和选型

**专项性能测试【必学】：**

- 前端性能测试（页面加载、渲染）
- 接口性能测试
- 数据库性能测试
- 微服务性能测试



### 学习建议

1）Gatling 是新一代性能测试工具，基于 Scala 开发，性能优秀。Gatling 使用代码定义测试场景，比 JMeter 的图形界面更灵活。Gatling 特别适合高并发场景，能够在单机上模拟数万并发用户。

![](https://pic.yupi.icu/1/68a74adbb7b95c0489ccafb1_header_home_V3.webp)

2）k6 是另一个现代化的性能测试工具，使用 JavaScript 编写脚本，对开发者友好。k6 支持云端测试，可以在全球多个地点同时发起测试。k6 的结果分析也很强大，支持实时监控。

3）选择性能测试工具要根据实际需求。JMeter 功能全面、社区活跃，适合大多数场景。Gatling 性能强、脚本灵活，适合高并发测试。k6 开发者友好、支持云端，适合现代化项目。

4）专项性能测试针对特定的测试对象。前端性能测试关注页面加载速度、渲染性能；接口性能测试关注接口响应时间、吞吐量；数据库性能测试关注查询性能、连接池等。要根据测试对象选择合适的工具和方法。

## 阶段 5：项目实战（20-35 天，仅供参考）

### 学习目标

通过完整的项目实践，综合运用所学知识，积累性能测试经验。



### 学习建议

1）选择一个实际项目进行性能测试。可以是公司的项目，也可以是开源项目。项目要有一定复杂度，包含多个接口、数据库操作等，这样才能体现性能测试的价值。

2）制定详细的性能测试方案。明确测试目标（如系统要支持多少并发用户、响应时间要在多少毫秒内）、测试场景、测试数据、测试环境等。测试方案要得到开发和运维的确认。

3）执行测试并分析结果。使用 JMeter 或 Gatling 执行性能测试，收集测试数据，分析性能指标。如果发现性能问题，要定位瓶颈，分析原因，提出优化建议。

4）参加性能优化。性能测试不应该是测试团队单独完成的工作，要和开发、运维协作。测试发现问题，开发优化代码，运维优化配置，共同提升系统性能。

5）编写性能测试报告。报告要清晰地说明测试场景、测试结果、性能瓶颈、优化建议等。报告是交付给业务和管理层的重要产出，要专业、准确、易懂。

## 阶段 6：求职备战（面试前 1 个月突击）

熟练掌握性能测试常见面试题，准备好简历和项目经历，顺利通过面试。

1. JMeter 的架构是怎样的？

1. 如何分析性能瓶颈？

3. 如何进行代码性能优化？
4. 缓存如何使用？有哪些注意事项？
5. 如何进行 JVM 调优？

1. 你做过哪些性能测试项目？详细说说。
