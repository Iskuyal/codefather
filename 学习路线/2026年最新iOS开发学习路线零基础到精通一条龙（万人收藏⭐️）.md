## 开篇介绍

iOS 是苹果公司开发的移动操作系统，运行在 iPhone、iPad 等设备上。iOS 开发使用 Swift 编程语言和苹果提供的开发框架（UIKit、SwiftUI 等）。

**为什么要学 iOS 开发？**

首先，iOS 开发工程师的薪资普遍高于 Android 开发，一线城市的 iOS 开发工程师平均薪资在 20-40K，高级 iOS 开发可以达到 40-60K。而且 iOS 开发的技术门槛相对较高，竞争相对较小，更容易找到高薪工作。

![](https://pic.yupi.icu/1/image-20251203111035126.png)

虽然 iOS 的市场份额不如 Android，但 iOS 用户的付费意愿和消费能力普遍更高，这使得 iOS 应用的商业价值往往超过 Android 应用。从王者荣耀到抖音、从支付宝到微信，几乎所有主流应用都有 iOS 版本。

从技术层面来看，iOS 开发的技术栈相对统一，不像 Android 那样有很多碎片化的设备和系统版本。iOS 的开发体验也更好，Xcode（苹果官方 IDE）功能强大、调试方便。而且苹果对开发者非常友好，提供了完善的文档、[WWDC 视频](https://developer.apple.com/videos/)、示例代码等学习资源。



### 学习前提

学习 iOS 开发有一些硬件和软件要求：

硬件要求（这点可能劝退了不少同学，包括大学时的鱼皮）：
- 必须有一台 Mac 电脑（MacBook、iMac、Mac Mini 等）
- iOS 开发只能在 macOS 上进行，无法在 Windows 或 Linux 上开发

软件要求：
- 安装 Xcode（苹果官方 IDE，免费）
- 建议使用最新版本的 macOS 和 Xcode

知识储备：
- 掌握至少一门编程语言（建议 Swift 或 Java）。关于 Swift 的详细学习，可以查看 [Swift 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754151050555394)
- 了解面向对象编程的基本概念
- 了解 Git 版本控制【建议】。关于 Git 的详细学习，可以查看 [Git 和 GitHub 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1789190804671012866)

## 整体学习建议

1）必须有 Mac 电脑！

这几乎是硬性要求，没有 Mac 就无法学习 iOS 开发。如果暂时买不起 Mac，可以先学习 Swift 语言（可以在 Linux 上运行），或者先学习 Android 开发。

2）虽然 Objective-C 是 iOS 的传统语言，但 Swift 是苹果官方推荐的首选语言，语法更现代、更安全。2025 年几乎所有新项目都使用 Swift，**建议直接学习 Swift**，不要在 Objective-C 上花太多时间。

3）SwiftUI 是 iOS 现代 UI 开发的标准，是未来趋势。虽然很多老项目还在使用 UIKit，但新项目越来越多地使用 SwiftUI。建议学完 iOS 基础后就学习 SwiftUI。

4）iOS 开发的学习必须结合项目实践。建议从简单的 App 开始（比如计算器、待办事项），逐步开发更复杂的应用（比如社交 App、电商 App）。在 AI 时代，也可以多多使用 [AI 资源导航](https://ai.codefather.cn/) 中的 AI 工具辅助开发。

5）苹果提供了非常丰富的学习资源，包括官方文档、WWDC 视频、示例代码等。建议多看 WWDC 视频，了解最新的技术和最佳实践。



## 阶段 1：Swift 编程语言（15-30 天，仅供参考）

Swift 是苹果在 2014 年推出的现代编程语言，语法简洁优雅，性能优秀，安全性高。Swift 的可选类型（Optional）是它的核心特性，能够在编译期避免空指针异常。



### 学习目标

掌握 Swift 编程语言，能够使用 Swift 进行基本的编程开发。



### 知识点

**Swift 基础语法【必学】：**

- 常量和变量（let、var）
- 基本数据类型（Int、Double、String、Bool）
- 运算符和表达式
- 控制流（if、switch、for、while）
- 可选类型（Optional）【重点】
- 元组（Tuple）

**函数和闭包【必学】：**

- 函数的定义和调用
- 参数和返回值
- 闭包（Closure）
- 尾随闭包（Trailing Closure）

**面向对象编程【必学】：**

- 类和结构体
- 属性（存储属性、计算属性）
- 方法
- 继承
- 构造器（Initializer）
- 析构器（Deinitializer）

**高级特性【建议学】：**

- 协议（Protocol）
- 扩展（Extension）
- 泛型（Generics）
- 枚举（Enum）
- 错误处理（Error Handling）
- ARC（自动引用计数）

**集合类型【必学】：**

- 数组（Array）
- 字典（Dictionary）
- 集合（Set）



### 学习重点

1）可选类型（Optional）是 Swift 的核心特性，用于处理值可能为空的情况。要深入理解可选类型的解包（!、?、if let、guard let）。

2）Swift 的协议（Protocol）类似于 Java 的接口，但功能更强大。协议是 Swift 编程的重要组成部分，要重点学习。

3）ARC（自动引用计数）是 Swift 的内存管理机制，要理解强引用、弱引用、无主引用的区别，避免循环引用导致的内存泄漏。

4）建议了解一下 Swift 和 Objective-C 的区别。Swift 语法更简洁、更安全、性能更好。如果看到老教程提到 Objective-C，可以跳过，重点学习 Swift。

5）这个阶段要多写代码，熟悉 Swift 的语法和特性。可以在 Xcode 的 Playground 中练习。

## 阶段 2：iOS 基础（10-30 天，仅供参考）

### 学习目标

掌握 iOS 开发的基础知识，能够开发简单的 iOS 应用。



### 知识点

**Xcode 开发环境【必学】：**

- Xcode 的安装和使用
- 项目的创建和运行
- 模拟器的使用
- 真机调试【建议学】
- Interface Builder（可选，SwiftUI 时代可以跳过）

**iOS 应用结构【必学】：**

- App 的生命周期
- UIApplication 和 AppDelegate
- SceneDelegate（iOS 13+）
- MVC 设计模式

**视图和控制器（UIKit）【必学】：**

- UIView：视图的基类
- UIViewController：视图控制器
- UILabel、UIButton、UIImageView 等基础控件
- UITableView 和 UICollectionView
- Auto Layout：自动布局

**导航和页面跳转【必学】：**

- UINavigationController：导航控制器
- UITabBarController：标签栏控制器
- Segue：页面跳转
- 模态视图（Modal）

**数据存储【必学】：**

- UserDefaults：轻量级存储
- 文件系统：File Manager
- Core Data：苹果官方数据库框架【建议学】
- SQLite【建议学】

**网络请求【必学】：**

- URLSession：网络请求
- JSON 解析（Codable）
- 图片加载和缓存
- 第三方库（Alamofire、Kingfisher）【建议学】



### 学习建议

1）Xcode 是 iOS 开发的官方 IDE，功能强大但学习难度较大。建议先熟悉 Xcode 的基本操作，比如创建项目、运行模拟器、调试代码等。

2）iOS 13+ 引入了 SceneDelegate，用于管理多窗口场景。但对于新手来说，可以先不用深入学习 SceneDelegate，重点学习 AppDelegate。

3）UITableView 是 iOS 开发中最常用的控件，用于显示列表数据。一定要熟练掌握 UITableView 的使用，包括自定义 Cell、数据刷新等。

4）Auto Layout 是 iOS 的自动布局系统，可以适配不同尺寸的设备。虽然学习曲线较陡，但非常重要，一定要掌握。

5）网络请求是 App 开发的核心功能，要学习如何使用 URLSession 进行网络请求，如何解析 JSON 数据。

## 阶段 3：SwiftUI 开发（10-30 天，仅供参考）

SwiftUI 是苹果在 2019 年推出的声明式 UI 框架，是 iOS 现代 UI 开发的标准和未来趋势。相比传统的 UIKit，SwiftUI 使用声明式语法，代码更简洁、开发效率更高。

![](https://pic.yupi.icu/1/try-again-button.png)



### 学习目标

掌握 SwiftUI 框架，能够使用 SwiftUI 快速开发 iOS 应用。



### 知识点

**SwiftUI 基础【必学】：**

- 声明式 UI 的概念
- View 和 ViewBuilder
- 基础控件（Text、Image、Button、TextField 等）
- 布局（VStack、HStack、ZStack、Spacer）
- 修饰符（Modifier）

**状态管理【必学】：**

- @State：视图的私有状态
- @Binding：状态的绑定
- @ObservedObject：观察对象
- @StateObject：状态对象
- @EnvironmentObject：环境对象

**列表和导航【必学】：**

- List：列表视图
- NavigationView 和 NavigationStack
- TabView：标签栏

**数据流【建议学】：**

- Combine 框架
- async/await：异步编程

**动画和手势【建议学】：**

- 动画（Animation）
- 转场（Transition）
- 手势（Gesture）



### 学习建议

1）SwiftUI 是声明式 UI 框架，和传统的命令式 UI（UIKit）完全不同。要转变思维方式，理解声明式编程的理念。

2）状态管理是 SwiftUI 的核心，要深入理解 @State、@Binding、@ObservedObject 等属性包装器的区别和使用场景。

3）SwiftUI 的学习曲线相对平缓，API 简洁优雅。建议多看官方示例，多写代码练习。

4）SwiftUI 还在快速发展中，每年的 iOS 新版本都会引入新特性。建议关注 WWDC 视频，了解最新的 SwiftUI 特性。

## 阶段 4：iOS 进阶（20-45 天，仅供参考）

GCD（Grand Central Dispatch）是苹果提供的多线程编程框架，用于处理并发任务。Combine 是苹果的响应式编程框架，用于处理异步事件流。掌握这些技术，能够开发出高性能、响应迅速的 iOS 应用。



### 学习目标

掌握 iOS 的进阶知识，能够开发复杂的 iOS 应用。



### 知识点

**多线程【必学】：**

- GCD（Grand Central Dispatch）
- DispatchQueue：主队列和后台队列
- async/await：异步编程（Swift 5.5+）
- Operation 和 OperationQueue【建议学】

**网络和数据【必学】：**

- URLSession 高级用法
- Combine 框架
- JSON 解析（Codable、JSONDecoder）
- 第三方网络库（Alamofire）【建议学】

**持久化【必学】：**

- UserDefaults
- FileManager
- Core Data【建议学】
- Realm 数据库【可不学】

**系统框架【建议学】：**

- Core Location：定位服务
- MapKit：地图
- Core Animation：动画
- AVFoundation：音视频
- PhotoKit：相册

**第三方库【建议学】：**

- CocoaPods：依赖管理工具
- Swift Package Manager：苹果官方依赖管理（推荐）
- 常用第三方库（Alamofire、Kingfisher、SnapKit 等）

**性能优化【建议学】：**

- 内存优化
- 启动优化
- 卡顿优化
- 网络优化
- Instruments 性能分析工具



### 学习建议

1）GCD 是 iOS 多线程编程的基础，要理解主队列和后台队列的区别。UI 操作必须在主线程进行，耗时操作应该放在后台线程。

2）async/await 是 Swift 5.5+ 引入的新特性，大大简化了异步编程。建议优先学习 async/await，而不是传统的回调方式。

3）Core Data 是苹果官方的数据库框架，功能强大但学习难度也较大。对于简单的应用，可以使用 UserDefaults 或 SQLite；对于复杂的应用，建议学习 Core Data。

4）Swift Package Manager 是苹果官方的依赖管理工具，正在逐渐取代 CocoaPods。建议优先学习 Swift Package Manager。

5）性能优化是 iOS 开发的重要工作，要学习如何使用 Instruments 分析应用的性能瓶颈。

## 写在最后

iOS 开发是一个技术门槛相对较高、但薪资待遇也相对较好的技术方向。虽然必须有 Mac 电脑才能学习，但一旦入门，你会发现 iOS 开发的体验非常好，技术栈也相对统一。

学习 iOS 开发建议优先学习 Swift 和 SwiftUI，这是 iOS 开发的未来方向。虽然很多老项目还在使用 UIKit，但新项目越来越多地使用 SwiftUI。掌握 SwiftUI，不仅能提高开发效率，还能让你在求职时更有竞争力。

在 AI 时代，iOS 平台对 AI 应用的支持越来越好。苹果推出了 Core ML、Create ML 等机器学习框架，可以在 iOS 设备上运行 AI 模型。

就讲到这里，希望这份学习路线能够帮助大家少走弯路，更快速地掌握 iOS 开发。加油各位未来的 IOS 大师们！
