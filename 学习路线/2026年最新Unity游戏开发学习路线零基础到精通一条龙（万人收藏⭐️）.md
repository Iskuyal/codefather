## 开篇介绍

Unity 是全球最流行的游戏引擎之一，以其易用性、跨平台能力和强大的功能而闻名，从独立游戏到 3A 大作，从手机游戏到主机游戏，Unity 都能胜任。

**为什么要学 Unity？**

Unity 是入门游戏开发的最佳选择。一方面 Unity 使用 C# 作为脚本语言，语法简洁易学，对新手非常友好。另外 Unity 的学习资源非常丰富，官方提供了大量教程、文档和示例项目，社区也非常活跃。而且 Unity 的个人版是完全免费的，你可以免费学习和开发游戏。

Unity 的核心优势是跨平台能力。使用 Unity 开发的游戏可以一键导出到 iOS、Android、Windows、macOS、PlayStation、Xbox、Nintendo Switch、WebGL 等 20 多个平台，真正实现了 “一次开发，到处运行”。

目前 Unity 的就业前景不错，随着游戏行业的快速发展，Unity 开发者的需求量一直在增长。Unity 不仅可以用于游戏开发，还可以用于 VR/AR 开发、数字孪生、元宇宙等新兴领域，职业发展空间很大。

### 学习前提

学习 Unity 游戏开发建议先掌握：
1. 编程基础：理解变量、函数、类、对象等基本概念
2. C# 语言基础：Unity 使用 C# 编写脚本【建议】

这里简单介绍一下 C# 语言吧。

C# 是微软开发的面向对象编程语言，语法简洁、功能强大。Unity 选择 C# 作为脚本语言，是因为 C# 的语法易学、性能优秀、生态丰富。如果你还不会 C#，可以先学习 C# 的基础语法（变量、函数、类等），也可以边学 Unity 边学 C#。关于 C# 的详细学习，可以查看 [C# 语言学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754042741043202)

## 整体学习建议

1）Unity 既支持 2D 也支持 3D 游戏开发，**建议从简单的 2D 游戏开始学习**，熟悉 Unity 的基本操作后，再学习 3D 游戏开发。

2）游戏开发的学习要以项目为导向，建议多做一些小游戏，如打砖块、贪吃蛇、跑酷、坤坤打篮球游戏等。每完成一个小游戏，都会有成就感，也能巩固所学知识。

3）Unity 官方提供了大量教程和示例项目，质量非常高。**建议多看官方教程，学习官方的最佳实践。**

4）Unity Asset Store 有大量免费和付费的资产（模型、音效、脚本、插件），可以大大提高开发效率。但不要过度依赖，核心逻辑还是要自己实现，否则面试时会露馅。



## 阶段 1：Unity 基础（15-25 天，仅供参考）

### 学习目标

掌握 Unity 编辑器的使用，理解 Unity 的基本概念。



### 知识点

**Unity 编辑器【必学】：**

- Unity Hub 和 Unity Editor 的安装
- Unity 界面（Scene、Game、Hierarchy、Project、Inspector）
- GameObject 和 Component
- Transform（位置、旋转、缩放）
- Prefab（预制体）
- 场景管理

**2D 游戏开发【必学】：**

- Sprite 和 Sprite Renderer
- 2D 碰撞器（Box Collider 2D、Circle Collider 2D）
- Rigidbody 2D（刚体）
- Tilemap（瓦片地图）

**输入系统【必学】：**

- Input 类（键盘、鼠标）
- Input System（新版输入系统）【建议学】

**摄像机【必学】：**

- Camera 组件
- 正交投影和透视投影
- 摄像机跟随



### 学习重点

1）建议安装较新版本的 Unity 6 进行学习。Unity Hub 是 Unity 的版本管理工具，可以方便地安装和切换 Unity 版本。

2）GameObject 是 Unity 中的基本对象，Component 是 GameObject 的组成部分。Unity 采用了组件化设计，通过组合不同的 Component 来实现功能。

3）Prefab 是 Unity 的预制体，可以将 GameObject 保存为 Prefab，方便复用和管理。

4）建议先学习 2D 游戏开发，熟悉 Unity 的基本操作后，再学习 3D 游戏开发。

5）这个阶段要多动手，尝试在 Unity 中创建简单的场景，添加对象，移动摄像机，熟悉 Unity 编辑器的使用。

## 阶段 2：C# 脚本编程（20-30 天，仅供参考）

### 学习目标

掌握 C# 在 Unity 中的使用，能够通过编写脚本控制游戏对象的行为和逻辑。



### 知识点

**C# 基础（Unity 中常用）【必学】：**

- 变量和数据类型
- 函数和方法
- 类和对象
- 继承和多态
- MonoBehaviour 基类

**Unity 生命周期【必学】：**

- Awake()、Start()
- Update()、FixedUpdate()、LateUpdate()
- OnEnable()、OnDisable()
- OnDestroy()

**组件操作【必学】：**

- GetComponent<>()：获取组件
- AddComponent<>()：添加组件
- Destroy()：销毁对象
- Instantiate()：实例化对象

**物体移动和旋转【必学】：**

- transform.position、transform.rotation
- transform.Translate()、transform.Rotate()
- Vector3 和 Quaternion

**碰撞和触发【必学】：**

- OnCollisionEnter()、OnCollisionStay()、OnCollisionExit()
- OnTriggerEnter()、OnTriggerStay()、OnTriggerExit()



### 学习重点

1）MonoBehaviour 是 Unity 脚本的基类，所有 Unity 脚本都要继承 MonoBehaviour。要理解 MonoBehaviour 的生命周期方法。

2）Update() 每帧调用一次，用于处理游戏逻辑；FixedUpdate() 以固定时间间隔调用，用于处理物理计算。

3）碰撞和触发是游戏中常用的功能，要理解 Collider 和 Trigger 的区别。Collider 会产生物理碰撞，Trigger 不会产生物理碰撞但会触发事件。

4）这个阶段要多写脚本，尝试控制游戏对象的移动、旋转、碰撞等，熟悉 Unity 的脚本编程。



### 学习资源

- [Unity 脚本编程官方文档](https://docs.unity.cn/cn/current/Manual/ScriptingSection.html)：官方文档
- [Unity C# 编程教程](https://unity.com/cn/how-to/programming-unity)：官方教程



## 阶段 3：物理系统（10-15 天，仅供参考）

### 学习目标

掌握 Unity 的物理系统，能够通过物理引擎实现重力、碰撞、刚体等真实的物理效果。



### 知识点

**2D 物理【必学】：**

- Rigidbody 2D：刚体
- Collider 2D：碰撞器
- 物理材质（Physics Material 2D）
- 关节（Joint 2D）【建议学】

**3D 物理【建议学】：**

- Rigidbody：刚体
- Collider：碰撞器（Box、Sphere、Capsule、Mesh）
- 物理材质（Physics Material）
- 射线检测（Raycast）

**力和速度【必学】：**

- AddForce()：施加力
- velocity：速度
- 重力和质量



### 学习建议

1）Rigidbody 是 Unity 物理系统的核心组件，添加 Rigidbody 后对象就会受物理引擎控制（如重力）。

![](https://pic.yupi.icu/1/Rigidbody2D.png)

2）物理计算应该在 FixedUpdate() 中进行，而不是 Update()，因为物理引擎以固定时间间隔更新。

3）射线检测可以用于实现射击、检测前方是否有障碍物等功能，非常实用。



## 阶段 4：动画和 UI（15-25 天，仅供参考）

### 学习目标

掌握 Unity 的动画系统和 UI 系统，能够制作动画和游戏界面。



### 知识点

**动画系统【必学】：**

- Animator 和 Animation
- 动画状态机（Animator Controller）
- 动画过渡（Transition）
- 动画参数（Parameter）
- 动画事件（Animation Event）【建议学】

**UI 系统【必学】：**

- Canvas：画布
- UI 组件（Text、Image、Button、Slider、InputField）
- 布局（Horizontal Layout、Vertical Layout、Grid Layout）
- UI 事件（点击、拖拽）
- UI 适配

**粒子系统【建议学】：**

- Particle System：粒子系统
- 特效制作（爆炸、火焰、烟雾）

**音频【建议学】：**

- Audio Source 和 Audio Listener
- 背景音乐和音效



### 学习建议

1）Animator Controller 是 Unity 的动画状态机，可以管理多个动画之间的切换。要理解状态和过渡的概念。

2）Canvas 是 UI 的容器，所有 UI 元素都要放在 Canvas 下。Canvas 有三种渲染模式（Screen Space、World Space、Camera），要理解各自的区别。

3）如果要开发移动端游戏，要特别注意 UI 适配，学习如何适配不同分辨率的屏幕。



## 阶段 5：Unity 进阶（15-25 天，仅供参考）

### 学习目标

掌握 Unity 的进阶特性，能够开发更复杂、更精美和流畅的游戏。



### 知识点

**3D 游戏开发【建议学】：**

- 3D 模型和材质
- 光照系统（Directional Light、Point Light、Spot Light）
- 地形系统（Terrain）
- 导航系统（NavMesh）

**协程【必学】：**

- Coroutine 的概念和使用
- yield return
- 异步操作

**数据持久化【必学】：**

- PlayerPrefs：简单存储
- JSON 序列化
- 文件读写

**性能优化【建议学】：**

- 对象池（Object Pool）
- 批处理（Batching）
- LOD（Level of Detail）
- Profiler：性能分析工具

**Shader 和特效【可不学】：**

- Shader Graph：可视化 Shader 编辑器
- 后处理效果（Post Processing）



### 学习建议

1）协程是 Unity 处理延迟操作的标准方式，如等待几秒后执行某个操作、播放动画序列等。要熟练掌握协程的使用。

2）对象池是性能优化的重要技术，可以避免频繁创建和销毁对象。在射击游戏中，子弹一般使用对象池管理。

3）Shader 相对复杂，对于新手来说可以先跳过，使用 Unity 自带的 Shader 即可。



## 阶段 6：项目实战（30-60 天，仅供参考）

### 学习目标

通过实际项目巩固所学知识，积累项目经验。



### 学习建议

1）从简单游戏开始：先开发一些经典的小游戏，如打砖块、贪吃蛇、跑酷游戏等，熟悉 Unity 的开发流程。

2）参考教程项目：跟着教程开发完整的游戏项目，学习游戏的设计和实现。

3）独立开发游戏：尝试独立开发一个游戏，从策划到开发到发布，体验完整的游戏开发流程。

4）发布游戏：完成游戏后，可以发布到 Steam、App Store、Google Play 等平台，体验游戏发行的过程。

5）参加 Game Jam：Game Jam 是限时游戏开发活动，可以锻炼快速开发能力，结识其他游戏开发者。

- 角色扮演游戏（RPG）

**官方教程项目：**
