## 开篇介绍

CI/CD 是持续集成（Continuous Integration）和持续交付/部署（Continuous Delivery/Deployment）的缩写。

其中，持续集成（CI）是指开发人员频繁地将代码集成到主分支，每次集成都通过自动化构建和测试来验证，及早发现问题。持续交付（CD）是指代码通过 CI 验证后，自动部署到测试环境或预发布环境。持续部署是持续交付的更进一步，自动部署到生产环境。

CI/CD 通过自动化构建、测试和部署流程，让软件能够快速、频繁、可靠地交付给用户，大大提高了软件开发的效率和质量。

**为什么要学 CI/CD？**

CI/CD 是 DevOps 的核心实践，是现代软件开发的标准流程。几乎所有互联网公司都实施了 CI/CD（没错，包括鱼皮的鱼厂），掌握 CI/CD 是开发工程师和运维工程师的必备技能。而且 CI/CD 可以大大提高开发效率，减少人工操作、降低出错率，让团队能够更快速地交付高质量软件。

CI/CD 的核心工具包括：Jenkins（老牌 CI/CD 工具）、GitLab CI（GitLab 内置）、GitHub Actions（GitHub 内置）、Travis CI、CircleCI 等。

## 整体学习建议

1）CI/CD 不仅是工具，更是一种文化和实践，要理解 CI/CD 如何提高软件交付效率和质量。

2）从简单开始，先搭建一个简单的 CI/CD 流水线（如代码提交后自动运行测试），熟悉基本流程。然后逐步增加复杂度（如自动部署、多环境部署等）。

3）选择合适的工具：Jenkins 是最成熟的工具，功能强大但配置复杂；GitLab CI 和 GitHub Actions 是现代工具，配置简单但功能也很强大。时至 2025 年，鱼皮建议 **优先学习 GitHub Actions 和 GitLab CI**，它们代表了 CI/CD 的未来方向，更适合新项目。

![](https://pic.yupi.icu/1/image-20251202002912697.png)

4）CI/CD 的学习一定要结合实践。建议为自己的项目搭建 CI/CD 流水线，体验自动化带来的便利。

5）推荐使用 **Serverless 部署平台**（如微信云托管、阿里云 Serverless），它们一般内置了 CI/CD 能力，代码提交后自动构建和部署，无需自己搭建流水线，非常方便。鱼皮的 [AI 超级智能体项目](https://www.codefather.cn/course/1915010091721236482) 就教大家如何使用微信云托管实现自动化部署，从代码提交到上线全自动。

5）AI 时代下，学习 CI/CD 的成本大大降低，可以用 AI 工具辅助编写配置文件、调试流水线。没事可以多到鱼皮的 [AI 资源大全](https://ai.codefather.cn/) 中看看，应该有不少对你有用的 AI 工具。

## 阶段 1：CI/CD 基础（3-12 天，仅供参考）

### 学习目标

理解 CI/CD 的核心概念和流程。

### 知识点

**CI/CD 基础概念【必学】：**

- 持续集成（CI）
- 持续交付（CD）
- 持续部署
- CI/CD 的流程

**CI/CD 的组成【必学】：**

- 代码仓库（Git）
- 构建工具（Maven、Gradle、npm 等）
- 测试工具
- 部署工具

**CI/CD 最佳实践【必学】：**

- 频繁集成
- 自动化测试
- 快速反馈
- 小步快跑



### 学习重点

1）CI/CD 的核心是自动化。每次代码提交都应该触发自动化流程：构建 → 测试 → 部署。

2）CI/CD 可以及早发现问题。如果测试失败，流水线会中断，避免有问题的代码进入生产环境。

3）要理解 CI、CD、持续部署的区别。CI 是自动构建和测试，CD 是自动部署到测试环境，持续部署是自动部署到生产环境。



## 阶段 2：GitHub Actions（7-20 天，仅供参考）


GitHub Actions 是 GitHub 官方的 CI/CD 工具，可以直接在代码仓库中配置自动化工作流，实现代码的自动构建、测试和部署。



### 学习目标

掌握 GitHub Actions，能够为 GitHub 项目搭建 CI/CD 流水线。

![](https://pic.yupi.icu/1/actions-quickstart-logs.png)



### 知识点

**GitHub Actions 基础【必学】：**

- Workflow 的概念
- .github/workflows 目录
- YAML 配置文件
- Events、Jobs、Steps

**Actions 市场【必学】：**

- 使用第三方 Actions
- 常用 Actions（checkout、setup-node、upload-artifact 等）

**高级特性【建议学】：**

- Matrix 策略（多版本测试）
- 缓存（Cache）
- Secrets（密钥管理）
- 环境（Environments）

**自托管 Runner【建议学】：**

- 自托管 Runner 的配置
- 和 GitHub 托管 Runner 的区别



### 学习建议

1）GitHub Actions 是 GitHub 内置的 CI/CD 工具，配置简单、功能强大。如果项目托管在 GitHub，强烈推荐使用 GitHub Actions。

2）Workflow 由一个或多个 Job 组成，Job 由多个 Step 组成，每个 Step 可以运行命令或使用 Action。

![](https://pic.yupi.icu/1/overview-actions-simple.png)

3）Actions 市场有大量第三方 Actions，可以直接使用，无需自己实现。比如 actions/checkout 用于检出代码，actions/setup-node 用于配置 Node.js 环境。

4）GitHub Actions 对公开仓库免费，私有仓库有免费额度。对于个人项目，免费额度一般就够用了。



### 学习资源

- ⭐ [GitHub Actions 官方文档](https://docs.github.com/zh/actions/get-started/quickstart)：最权威的学习资料，而且还有中文版，很贴心~
- [GitHub Actions 持续集成文档](https://docs.github.com/zh/actions/get-started/continuous-integration)
- [GitHub Actions 持续部署文档](https://docs.github.com/zh/actions/get-started/continuous-deployment)



## 阶段 3：GitLab CI/CD（5-15 天，仅供参考）


GitLab CI/CD 是 GitLab 内置的持续集成工具，通过配置文件即可实现强大的自动化流程。



### 学习目标

掌握 GitLab CI/CD，能够为 GitLab 项目搭建 CI/CD 流水线。



### 知识点

**GitLab CI 基础【必学】：**

- .gitlab-ci.yml 配置
- Pipeline、Stage、Job
- GitLab Runner

**常用配置【必学】：**

- script、before_script、after_script
- variables（变量）
- cache 和 artifacts
- only 和 except（触发条件）

**Runner【必学】：**

- Shared Runner
- Specific Runner
- Runner 的安装和注册



### 学习建议

1）GitLab CI 是 GitLab 内置的 CI/CD 工具，配置文件是 `.gitlab-ci.yml`。如果项目托管在 GitLab，GitLab CI 是首选。

![](https://pic.yupi.icu/1/three_stages_v13_6.png)

2）GitLab CI 的 Pipeline 由多个 Stage 组成，每个 Stage 包含多个 Job，Job 在 Runner 上执行。

![](https://pic.yupi.icu/1/pipeline_graph_v17_9.png)

3）GitLab 提供了免费的 Shared Runner，可以直接使用。如果需要更多资源或特定环境，可以自己搭建 Runner。



### 学习资源

- [GitLab CI 官方文档](https://docs.gitlab.com/topics/build_your_application/)：官方文档
- [GitLab CI vs GitHub Actions](https://www.bytebase.com/blog/gitlab-ci-vs-github-actions/)：对比



## 阶段 4：Jenkins（10-15 天，仅供参考）


Jenkins 是经典的开源 CI/CD 工具，拥有丰富的插件生态，适合构建复杂的企业级自动化流程。



### 学习目标

掌握 Jenkins，理解传统 CI/CD 工具的使用。



### 知识点

**Jenkins 基础【必学】：**

- Jenkins 的安装和配置
- Pipeline 流水线
- Jenkinsfile

**插件系统【建议学】：**

- 常用插件
- 插件的安装和配置

**高级特性【建议学】：**

- 参数化构建
- 多分支流水线
- Blue Ocean【可不学】



### 学习重点

1）Jenkins 是老牌 CI/CD 工具，功能强大但配置复杂。虽然 GitHub Actions 和 GitLab CI 更现代，但很多企业仍在使用 Jenkins，了解 Jenkins 仍然很重要。

Jenkins 的图标是个老爷爷，跟 KFC 有异曲同工之妙~

![](https://pic.yupi.icu/1/Jenkins-featured.png.webp)

2）Jenkins Pipeline 可以用 Groovy 脚本定义，非常灵活。Jenkinsfile 是 Pipeline 的配置文件，可以版本化管理。

3）Jenkins 的插件生态非常丰富，几乎任何功能都有对应的插件。但插件过多也导致管理复杂。

## 阶段 5：项目实战（10-30 天，仅供参考）

### 学习目标

通过实际项目巩固所学知识，积累 CI/CD 项目经验。



### 学习建议

1）搭建完整的 CI/CD 流水线：从代码提交到自动测试、自动部署、监控告警，体验完整流程。

2）多环境部署：实现开发环境、测试环境、生产环境的自动化部署。

3）集成自动化测试：在 CI/CD 流水线中集成单元测试、集成测试、E2E 测试等。

4）优化流水线：包括但不限于优化构建速度（如使用缓存）、提高可靠性（如失败重试）等等。



### 项目推荐

**传统部署场景：**

- 前端项目 CI/CD（构建、测试、部署到 CDN）
- 后端项目 CI/CD（构建、测试、部署到服务器）
- Docker 镜像 CI/CD（构建镜像、推送到镜像仓库）
- Kubernetes 应用 CI/CD（部署到 K8s 集群）

**Serverless 部署场景（推荐）：**

- 微信云托管：Serverless 部署平台，支持 CI/CD 流水线，无需管理服务器，自动弹性伸缩
- 阿里云 Serverless、腾讯云 Serverless 等云平台
