## 开篇介绍

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时环境，让 JavaScript 可以运行在服务器端。在 Node.js 出现之前，JavaScript 只能运行在浏览器中，主要用于前端开发。Node.js 的出现打破了这个限制，使得 JavaScript 可以用于后端开发、命令行工具、桌面应用等多种场景。

Node.js 的核心特点是事件驱动、非阻塞 I/O 模型，这使得 Node.js 非常适合处理高并发的 I/O 密集型应用，如 Web 服务器、实时通信、API 网关等。Node.js 使用单线程处理请求，但通过事件循环机制实现异步 I/O，能够同时处理成千上万的并发连接。

**为什么要学 Node.js？**

对于前端开发者来说，学习 Node.js 可以让你成为全栈工程师，既能开发前端界面，又能开发后端接口，大大提升你的竞争力。对于后端开发者来说，Node.js 是一个轻量级、高性能的后端技术选择，特别适合快速开发中小型应用。

💡 如果学过 Java 后端再学 Node 后端，可以说是轻而易举了。

Node.js 的生态系统非常丰富，npm（Node Package Manager）是全球最大的开源库生态系统，拥有数百万个开源包。无论你想实现什么功能，几乎都能找到现成的 npm 包，大大提高了开发效率。

![](https://pic.yupi.icu/1/image-20251128183935610.png)

必须要提的是，Node.js 也可以用于 AI 应用开发。虽然 AI 模型训练一般使用 Python，但 Node.js 可以用于 AI 应用的 API 开发、数据处理、前端交互等，是构建 AI 应用较为流行的方式。

## 整体学习建议

1）掌握 JavaScript 是前提：Node.js 使用 JavaScript 语言，如果 JavaScript 基础不牢固，学习 Node.js 会很吃力。建议先系统学习 JavaScript，特别是 ES6+ 的新特性。

2）理解异步编程：Node.js 的核心是异步非阻塞 I/O，要深入理解异步编程的概念。从回调函数到 Promise，再到 async/await，是一个循序渐进的过程。

3）多做项目实践：Node.js 的学习必须结合项目实践。建议从简单的 HTTP 服务器开始，逐步开发 RESTful API、Web 应用、实时聊天等项目。

4）善用 npm 生态：npm 有数百万个开源包，学会使用 npm 包可以大大提高开发效率。但也要避免过度依赖第三方包，有些核心逻辑还是要自己实现。

5）学习 Node.js 时建议多用 AI 工具辅助开发，如生成代码、调试错误、理解概念等，有时间可以多到鱼皮的 [AI 资源大全](https://ai.codefather.cn/) 中找找工具、看看资讯。

## 阶段 1：Node.js 基础（8-15 天，仅供参考）

### 学习目标

理解 Node.js 的基本概念，掌握 Node.js 的安装和基本使用。



### 知识点

**Node.js 基础概念【必学】：**

- Node.js 的定义和特点
- Node.js 的应用场景
- 事件驱动和非阻塞 I/O
- 单线程模型
- 事件循环（Event Loop）

**安装和配置【必学】：**

- Node.js 的安装（官网下载、nvm 管理版本）
- npm 的使用（安装、卸载、更新包）
- package.json 文件
- node_modules 目录

**模块系统【必学】：**

- CommonJS 模块（require、module.exports）
- ES Module（import、export）
- 内置模块和第三方模块
- 自定义模块

**npm 包管理【必学】：**

- npm init：初始化项目
- npm install：安装依赖
- npm scripts：脚本命令
- package-lock.json
- yarn 和 pnpm【建议学】



### 学习建议

1）建议使用 nvm（Node Version Manager）来管理 Node.js 版本，方便在不同项目之间切换 Node.js 版本。可以看 [鱼皮的前端 NVM 安装视频教程](https://www.bilibili.com/video/BV14SUNYREv8)。

2）CommonJS 是 Node.js 的传统模块系统，ES Module 是 JavaScript 的标准模块系统。Node.js 现在同时支持两种模块系统，建议优先学习 ES Module。

3）npm 是 Node.js 的默认包管理器，但 yarn 和 pnpm 在某些方面更优秀（如速度、磁盘占用），建议了解 yarn 和 pnpm。

4）这个阶段要 **多动手实践**，创建简单的 Node.js 脚本，熟悉 Node.js 的基本操作。

## 阶段 2：Node.js 核心模块（8-15 天，仅供参考）

### 学习目标

掌握 Node.js 的核心模块，能够使用 Node.js 进行文件操作、网络请求等基本开发。



### 知识点

**文件系统（fs）【必学】：**

- 读取文件（readFile、readFileSync）
- 写入文件（writeFile、writeFileSync）
- 文件信息（stat）
- 目录操作（mkdir、readdir）
- 文件流（Stream）

**路径处理（path）【必学】：**

- path.join()：路径拼接
- path.resolve()：绝对路径
- path.dirname()、path.basename()、path.extname()

**HTTP 模块【必学】：**

- 创建 HTTP 服务器
- 处理请求和响应
- 路由处理
- 静态文件服务

**事件（events）【必学】：**

- EventEmitter
- on()：监听事件
- emit()：触发事件
- once()、off()

**流（Stream）【建议学】：**

- 可读流（Readable）
- 可写流（Writable）
- 管道（pipe）

**Buffer【建议学】：**

- Buffer 的创建和使用
- Buffer 和字符串的转换

**其他核心模块【建议学】：**

- url：URL 解析
- querystring：查询字符串解析
- crypto：加密
- util：工具函数



### 学习重点

1）fs 模块是 Node.js 最常用的核心模块之一，要熟练掌握文件的读写操作。同步方法（如 readFileSync）会阻塞代码执行，建议使用异步方法。

2）HTTP 模块是 Node.js 的核心，可以用来创建 Web 服务器。虽然实际开发中更推荐使用 Express、Koa 等框架，但理解 HTTP 模块的工作原理非常重要。

3）事件驱动是 Node.js 的核心特性，要理解 EventEmitter 的工作原理。Node.js 的很多模块都继承自 EventEmitter。

4）Stream 是 Node.js 处理大文件的利器，可以边读边处理，避免一次性加载整个文件到内存。

## 阶段 3：Web 框架（10-20 天，仅供参考）


Web 框架包括 Express、Koa 等，提供了快速构建 Web 应用的工具和中间件，是 Node.js 后端开发的核心。



### 学习目标

掌握 Express 或 Koa 框架，能够快速开发 RESTful API。



### 知识点

**Express 框架【必学】：**

- Express 的安装和使用
- 路由（Router）
- 中间件（Middleware）
- 请求和响应对象
- 模板引擎（EJS、Pug）【可不学】
- 错误处理
- 静态文件服务

**Koa 框架【建议学】：**

- Koa 的特点（洋葱模型）
- Koa 中间件
- Koa 路由（koa-router）
- 错误处理

**RESTful API 设计【必学】：**

- RESTful API 的概念和规范
- HTTP 方法（GET、POST、PUT、DELETE）
- 状态码
- 参数传递（Query、Body、Params）
- API 文档（Swagger、Postman）

**身份认证【必学】：**

- Session 和 Cookie
- JWT（JSON Web Token）
- OAuth 2.0【建议学】

**文件上传【建议学】：**

- Multer 中间件
- 文件存储（本地存储、云存储）



### 学习建议

1）Express 是 Node.js 最流行的 Web 框架，简单易学，社区活跃。建议优先学习 Express。

2）Koa 是 Express 团队开发的新一代框架，采用 async/await 语法，代码更简洁。如果时间充足，建议学习 Koa。

3）中间件是 Express/Koa 的核心概念，要理解中间件的执行顺序和洋葱模型。

4）RESTful API 是现代 Web 开发的标准，要理解 RESTful 的设计规范，如使用 HTTP 方法表示操作、使用 HTTP 状态码表示结果等。

5）JWT 是现代 Web 应用最常用的认证方式，要理解 JWT 的工作原理和使用方法。

## 阶段 4：数据库操作（10-15 天，仅供参考）

### 学习目标

数据库操作是后端开发的必备技能，需要掌握如何在 Node.js 中连接和操作 MySQL、MongoDB 等数据库，能够进行数据的增删改查。



### 知识点

**MySQL 操作【必学】：**

- mysql2 库
- 连接数据库
- 执行 SQL 语句
- 参数化查询（防止 SQL 注入）
- 连接池

**MongoDB 操作【建议学】：**

- mongoose 库
- 定义 Schema 和 Model
- CRUD 操作
- 查询和聚合

**ORM 框架【建议学】：**

- Sequelize：MySQL ORM
- Prisma：现代 ORM 框架（推荐）
- TypeORM

**Redis 操作【建议学】：**

- ioredis 库
- 基本操作（set、get、del）
- 缓存应用



### 学习建议

1）Node.js 操作数据库一般使用第三方库。MySQL 推荐使用 mysql2，MongoDB 推荐使用 mongoose。

2）ORM 框架可以简化数据库操作，避免编写 SQL 语句。Prisma 是近年来非常流行的 ORM 框架，语法简洁、类型安全，建议学习。鱼皮当年也用它做过项目。

## 阶段 5：项目实战（20-30 天，仅供参考）

### 学习建议

1）从简单项目开始：先开发一些简单的项目，如 Todo API、博客系统 API、用户管理系统等。

4）应用所学技术：在项目中应用 Express/Koa、数据库、身份认证、文件上传等技术。

5）部署上线：完成项目后，可以部署到服务器上（如阿里云、腾讯云），体验完整的项目上线流程。

## 写在最后

Node.js 是一个轻量级、高性能的后端技术，特别适合快速开发和微服务架构。对于前端开发者来说，学习 Node.js 可以让你成为全栈工程师，大大提升你的竞争力。

学习 Node.js 的关键是理解异步编程和事件驱动模型。虽然这些概念比较抽象，但只要多做实践，多写代码，就能逐渐掌握。
