## 开篇介绍

Rust 是 Mozilla 研究院于 2010 年开始开发的系统编程语言，于 2015 年发布 1.0 版本。Rust 的设计目标是安全、并发、实用，通过创新的所有权系统在编译期保证内存安全，同时提供接近 C/C++ 的性能。Rust 被称为 “没有垃圾回收的内存安全语言”，解决了 C/C++ 的内存安全问题，又避免了 Java/Go 的垃圾回收性能开销。

Rust 的核心特性是所有权系统（Ownership System），这是 Rust 最独特也是最难的部分。通过所有权、借用、生命周期等概念，Rust 在编译期就能发现内存安全问题（如空指针、悬垂指针、数据竞争），而不需要运行时检查。这使得 Rust 程序既安全又高效。

**为什么要学 Rust？**

Rust 是近年来最受欢迎的编程语言之一，连续多年在 Stack Overflow 开发者调查中被评为 “最受喜爱的编程语言”。越来越多的公司开始使用 Rust，比如 Amazon、微软、Google、Meta 等科技巨头都在生产环境中使用 Rust。Linux 内核也开始支持 Rust，这标志着 Rust 在系统编程领域的重要地位。

![](https://pic.yupi.icu/1/rust.jpg)

Rust 的应用场景非常广泛，包括系统编程（操作系统、驱动程序）、网络服务（Web 服务器、微服务）、嵌入式开发、区块链开发、WebAssembly、命令行工具等。Rust 的性能接近 C/C++，但安全性远超 C/C++，是系统编程的理想选择。

💡 关于 WebAssembly 的详细学习，可以查看 [WebAssembly 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990754412812873730)。

在 AI 时代，Rust 也开始进入 AI 领域。一些 AI 框架和工具开始使用 Rust 开发，如 Hugging Face 的 tokenizers 库、Polars 数据处理库等。Rust 的高性能和内存安全使其成为 AI 基础设施的优秀选择。



### 学习前提

学习 Rust 建议先掌握：
1. 编程基础：至少会一门编程语言（C/C++、Java、Python 等）

3. 系统编程概念：了解编译、链接、内存管理等【建议】

Rust 的学习曲线相对陡峭，特别是所有权系统对新手来说比较难理解。**建议有一定编程基础后再学习 Rust。**

## 整体学习建议

1）所有权系统是核心：所有权系统是 Rust 最独特也是最难的部分，要花时间深入理解。不要着急，慢慢理解所有权、借用、生命周期的概念。

2）多看官方文档：[Rust 的官方文档《The Rust Programming Language》](https://doc.rust-lang.org/book/)（也叫 Rust 圣经）写得非常好，[中文翻译](https://kaisery.github.io/trpl-zh-cn/) 质量也很高。强烈建议阅读官方文档。

![](https://pic.yupi.icu/1/image-20251128212943741.png)

什么？你说你看不懂官方文档？

那我建议你还是先别学 Rust 了，先学其他语言沉淀沉淀。

3）多写代码：Rust 的编译器非常严格，一开始可能会遇到很多编译错误。不要气馁，这些错误都是在帮你写出更安全的代码。多写代码，多看编译器的错误提示。

4）善用 Rust 编译器：Rust 的编译器错误提示非常友好，会告诉你哪里出错了，为什么出错，如何修复。要仔细阅读编译器的提示。

5）善用 AI 工具：学习 Rust 时可以多用 AI 工具（比如 Gemini、DeepSeek）辅助理解概念、调试错误。平时没事儿可以多看看 [AI 资源大全](https://ai.codefather.cn/) 中的工具。

## 阶段 1：Rust 基础语法（15-20 天，仅供参考）

### 学习目标

掌握 Rust 的基础语法，能够编写简单的 Rust 程序。



### 知识点

**基础语法【必学】：**

- 变量和可变性（let、mut）
- 数据类型（整数、浮点数、布尔、字符、字符串）
- 函数
- 控制流（if、loop、while、for）
- 注释和文档注释

**所有权系统【必学，核心重点】：**

- 所有权（Ownership）
- 移动（Move）和克隆（Clone）
- 引用和借用（References & Borrowing）
- 可变引用和不可变引用
- 悬垂引用

**切片【必学】：**

- 字符串切片（&str）
- 数组切片（&[T]）



### 学习建议

1）所有权是 Rust 的核心概念，也是最难的部分。要理解 “一个值在任意时刻只能有一个所有者” 的规则。

2）借用是 Rust 的重要概念，可以让你在不获取所有权的情况下使用值。要理解不可变借用和可变借用的规则。

3）Rust 的编译器非常严格，会在编译期检查所有权规则。一开始可能会遇到很多编译错误，不要气馁，这是正常的。

4）建议在 [Rust Playground](https://play.rust-lang.org/) 在线编辑器中练习 Rust 代码。

## 阶段 2：结构体、枚举和模式匹配（10-15 天，仅供参考）

### 学习目标

掌握 Rust 的结构体、枚举和模式匹配，能够轻松组织数据和控制流程。



### 知识点

**结构体【必学】：**

- 结构体的定义和使用
- 元组结构体
- 结构体方法（impl）
- 关联函数

**枚举【必学】：**

- 枚举的定义和使用
- Option<T>：可选值
- Result<T, E>：错误处理

**模式匹配【必学】：**

- match 表达式
- if let 和 while let
- 模式的类型



### 学习重点

1）`Option<T>` 和 `Result<T, E>` 是 Rust 的核心类型，用于处理可能为空的值和错误。要熟练使用这两种类型。

2）模式匹配是 Rust 的强大特性，可以让代码更简洁、更安全。match 表达式必须穷尽所有可能。

3）Rust 的枚举比其他语言更强大，可以关联数据，类似于代数数据类型（ADT）。



### 学习资源

- [Rust 结构体和枚举](https://kaisery.github.io/trpl-zh-cn/)：官方文档



## 阶段 3：泛型、trait 和生命周期（15-20 天，仅供参考）

### 学习目标

掌握 Rust 的泛型、trait 和生命周期。



### 知识点

**泛型【必学】：**

- 泛型函数和泛型结构体
- 泛型枚举
- 泛型约束

**trait【必学】：**

- trait 的定义和实现
- trait 作为参数
- trait bound
- trait 对象

**生命周期【必学，难点】：**

- 生命周期的概念
- 生命周期注解（'a）
- 生命周期省略规则
- 静态生命周期（'static）



### 学习建议

1）trait 类似于其他语言的接口，但功能更强大。trait 是 Rust 实现多态的方式。

2）生命周期是 Rust 的难点，用于描述引用的有效范围。生命周期注解看起来很复杂，但实际上大部分时候编译器可以自动推断，不需要手动标注。

3）这个阶段的内容相对高级，需要反复学习和练习才能掌握，刚开始看不懂也不用灰心。



### 学习资源

- [Rust 泛型和 trait](https://kaisery.github.io/trpl-zh-cn/)：官方文档



## 阶段 4：并发编程（10-15 天，仅供参考）

### 学习目标

掌握 Rust 的并发编程，理解 Rust 如何保证线程安全，避免数据竞争。



### 知识点

**线程【必学】：**

- 创建线程（thread::spawn）
- 线程间通信（通道 channel）
- 共享状态（Arc、Mutex）

**并发安全【必学】：**

- Send 和 Sync trait
- 数据竞争的避免

**异步编程【建议学】：**

- async/await
- Future trait
- Tokio 异步运行时



### 学习建议

1）Rust 的并发安全是编译期保证的，编译器会检查数据竞争。要理解 Send 和 Sync trait 的作用。

2）异步编程是 Rust 的重要特性，Tokio 是 Rust 最流行的异步运行时，用于开发高性能网络服务。



### 学习资源

- [Rust 并发编程](https://kaisery.github.io/trpl-zh-cn/)：官方文档
- [Tokio 官方教程](https://tokio.rs/tokio/tutorial)：异步编程



## 阶段 5：项目实战（20-40 天，仅供参考）

### 学习目标

通过实际项目巩固所学知识，积累项目经验。

### 学习建议

1）从命令行工具开始：Rust 非常适合开发命令行工具，建议从简单的 CLI 工具开始练习。

2）开发 Web 服务：使用 Actix-web 或 Rocket 框架开发 Web 服务，学习 Rust 的 Web 开发。

3）系统编程：尝试用 Rust 开发一些系统工具，如文件系统、网络工具等。

4）阅读优秀代码：GitHub 上有很多优秀的 Rust 项目，阅读源码可以学习 Rust 的最佳实践。

- 命令行工具（grep、cat、find 的 Rust 实现）

- 简单的数据库
