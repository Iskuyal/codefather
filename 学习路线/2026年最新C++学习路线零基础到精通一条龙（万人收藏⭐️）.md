## 开篇介绍

C++ 是一门拥有几十年历史的经典编程语言，从诞生之初就以其高性能、灵活性和强大的系统控制能力而闻名。它是 C 语言的扩展，支持面向对象编程、泛型编程、过程式编程等多种编程范式，因此具有很高的灵活性和可扩展性。

**为什么要学 C++ 呢？**

虽然 C++ 相对于 Java、Python 等高级语言来说更加底层、学习曲线更陡峭，但正是因为它更接近硬件、能够精细控制系统资源，所以在追求极致性能的领域（如游戏引擎、音视频处理、高频交易、操作系统、嵌入式系统等）中，C++ 依然是不可替代的首选语言。

**学好 C++，你能做什么呢？**

你可以开发高性能服务器、游戏引擎、音视频处理系统、嵌入式设备程序、量化交易系统、甚至是操作系统本身，应用场景非常广泛且极具技术含量。

C++ 标准语法的发展也非常快，目前新的标准已经到 C++ 23 了。而且随着它的发展，它具有了丰富的语法和库支持。大学计算机专业的同学一般也会从 C++（C 语言）开始学起，因为它能帮助你深入理解计算机底层原理。

很多 AI 框架的底层（如 TensorFlow、PyTorch 的核心部分）都是用 C++ 实现的，因为只有 C++ 能够提供足够的性能来支撑大规模的 AI 计算。掌握 C++ 和 AI 应用开发，将会拥有更强的竞争力。

### 就业方向

C++ 的就业方向还是非常多的，鱼皮建议大家首先根据自己的兴趣爱好来做选择。

1）服务端开发（后端开发）：C++ 语言在高性能服务器开发、大数据处理、金融领域等都有广泛应用。对于用户量和并发量非常高的系统（比如微信后台、游戏服务器），一般会选择更高性能的 C++ 来开发。服务端开发工程师除了要学习后端通用知识（比如数据库、缓存），还需要更熟悉网络编程、并发编程、内存管理等技术。

2）系统开发：高性能的 C++ 无疑是系统开发的王牌语言！系统开发工程师负责设计和实现操作系统、驱动程序、嵌入式系统、网络协议栈等系统级软件。从事这个岗位必须熟练掌握 C++ 的语法、各种数据结构和算法、多线程编程、Linux 操作系统等技术。

3）客户端开发：一般是指开发用户看得到的界面。C++ 在桌面应用程序、移动应用程序和游戏客户端开发中有着广泛应用。C++ 客户端开发工程师和前端开发有着相似的学习要求，都需要掌握 GUI 编程、布局知识、跨平台开发技术。现在主流的开发框架是 Qt（鱼皮的毕设就是用它做的）。

4）嵌入式开发：C++ 被广泛应用于开发网络设备、汽车电子、智能家居、医疗设备等产品。嵌入式开发工程师和系统开发工程师的学习路径有相似之处，都需要学好 C++ 系统级编程以及操作系统知识。此外，嵌入式开发还需要了解硬件、嵌入式开发平台和相关工具。

5）游戏开发：C++ 是游戏开发的主语言之一（另一个是 C#），比如当下很火的虚幻引擎（Unreal Engine）的主语言就是 C++！对于游戏开发工程师，最重要的是热爱游戏！游戏开发也分客户端和服务端开发，前者需要熟悉游戏引擎、图形学、物理引擎等技术，后者和服务端开发的要求差不多，都要学好网络编程、并发编程。

- [游戏服务端开发学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755966940917761)

6）音视频处理：C++ 在音视频编解码、视频渲染、直播推流等领域有着非常广泛的应用！音视频处理工程师除了 C++ 基础知识外，还需要熟悉音视频格式、编解码技术（如 H.264、H.265）、FFmpeg 等相关技术。

7）图像处理：和音视频处理类似，都属于 C++ 计算类工作。图像处理工程师需要熟悉图像处理算法、计算机视觉、OpenCV 等，掌握各种 C++ 的图像处理库和相关算法。这类工作对数学会有一定要求，有些公司可能把它们定义为"算法工程师"。

8）量化交易开发：金融领域对交易系统的性能要求极高，C++ 是量化交易系统开发的首选语言。量化交易工程师需要掌握 C++ 高性能编程、金融知识、数学建模等技能。

9）SDK 开发：C++ 也适用于开发各种 SDK（软件开发工具包）和基础库。SDK 开发工程师需要熟悉 C++ 的语言特性、API 设计、掌握软件开发原则和设计模式，要能够设计出易用、高性能、可靠的接口，提供给其他开发者使用。

---

从上面这些岗位我们会发现，虽然都是学 C++，但不同方向的工程师需要侧重掌握的技能和专业知识也是有区别的。但是如果你还没确定选什么具体的方向，**不妨先掌握它们都要求的通用技能**，即能够熟练使用 C++ 语法来编程，打好基础。



### 选 C++ 还是 Java？

这是很多同学纠结的问题，鱼皮来给大家分析一下。

首先，明确一点，C++ 和 Java 都是非常优秀的主流编程语言，编程语言和技术没有绝对的优劣之分，只不过在不同的场景下有更优的选择。而且，像很多大神说的一样，语言只是工具，编程思想才是核心。

如何选择呢？

- 如果你对操作系统底层、嵌入式、图像处理、音视频处理、游戏开发、高性能计算等方向感兴趣，建议选择 C++。
- 如果你更喜欢开发业务系统（如电商平台、管理系统）、快速实现功能、思考和实现业务逻辑，就选择 Java。

简单来说，C++ 更侧重于 “如何让程序跑得更快、更省资源”，而 Java 更侧重于 “如何快速开发出可靠的业务系统”。

对于身在校园、且时间充裕的同学来说，鱼皮更建议 “我全都要”。可以先学 C 语言或 C++ 打基础（这也是大部分计算机专业学生在高校内学到的第一门编程语言），因为它相对底层，能帮助你接触到更多计算机的概念，比如内存管理、指针、操作系统资源管理、网络通讯等，从而为学习其他编程语言做铺垫。然后再学习 Java 来快速做项目、找工作。

- 关于 C 语言的详细学习，可以查看 [C 语言学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990753999116087297)
- 关于 Java 的详细学习，可以查看 [Java 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1789190431398928386)

此外，因为 Java 和 C++ 都可以作为后端开发语言，所以无论把哪门语言作为主语言，**都要重点学好后端开发通用知识**，比如数据库、缓存、消息队列、Socket 通讯、Linux、Git、计算机基础知识等。但选择 C++ 的同学要对操作系统、网络和 Linux 更熟悉一些。

## 整体学习建议

1）基础一定要打牢：这是 C++ 相对于 Java 这种上层语言来说最重要的建议！因为 C++ 可以更灵活地操作系统（比如自主分配内存、直接操作指针），所以如果基础不牢固，你不仅不能发挥出 C++ 的高性能等特性，反而还可能写出性能更差、甚至会崩溃的程序。像 C++ 基础特性（指针、引用、内存管理等）、操作系统、计算机网络、算法数据结构等计算机基础都要认真去学。

2）多动手实践：学 C++ 不能只看不练。一定要多练习语法、多敲代码、多实践做项目。C++ 的很多概念（比如指针、引用、内存管理）光看理论很难理解，必须自己动手写代码、调试程序才能真正掌握。

3）在 AI 时代，学习 C++ 可以更高效！遇到编译错误可以让 AI 帮你分析、遇到不理解的概念可以问 AI，还可以让 AI 帮你生成一些模板代码。推荐使用鱼皮的 [AI 资源大全](https://ai.codefather.cn/)，里面有各种好用的 AI 编程工具。但是记住，AI 是辅助工具，基础知识还是要自己扎实掌握。

4）多读优秀代码：C++ 有很多优秀的开源项目和库（如 STL、Boost、muduo 等），多读别人的代码、学习别人是怎么设计和实现的，能够快速提升你的编程能力。

5）持续学习：C++ 标准一直在更新（C++ 11、14、17、20、23），新特性层出不穷。要保持学习的习惯，关注 C++ 社区的最新动态。

## 阶段 2：C++ 进阶（30-90 天，仅供参考）

上个阶段我们主要通过看视频来快速入门 C++ 基础编程语法；而这个阶段要做的是巩固基础、学习更多 C++ 的特性、甚至是底层原理，建议大家通过阅读经典书籍的方式来学习。



### 学习建议

1）通过书籍系统学习：视频教程适合快速入门，但是要系统掌握 C++ 的特性和细节，还是要看书。先系统学习 C++ 基础及进阶语法，可以读《C++ Primer Plus》和《Effective C++》这两本经典书籍。

2）学习顺序很重要：学到这里，建议先去学习本路线阶段 3 的计算机基础，否则后面的书（比如《STL 源码剖析》《Linux 高性能服务器编程》）啃起来会很累。

### MySQL 数据库

关于 MySQL 数据库的详细学习，可以查看 [MySQL 数据库学习路线](https://www.codefather.cn/course/1789189862986850306/section/1789190581420793858)

数据库就是存放数据的库，负责存储、管理和检索数据。

我们平时看到的网站上的数据，基本都是存放到数据库中的，在用户需要的时候，再通过后端程序从数据库中取出来。

数据库又分为关系型数据库和非关系型数据库。关系型数据库适用于存储相互之间有关联的数据，比如做一个学生管理系统，要存储学生、班级，一个班级有多个学生，这就是一种关联关系，关系型数据库典型的技术是 MySQL；而非关系型数据库主要用于存储零散的、相互独立的数据，典型的技术是 Redis。

此处我们主要学习的是关系型数据库，并且以最主流、学习成本相对更低的关系型数据库 MySQL 为主。

对 C++ 同学来说，对数据库的学习包括以下 4 个方面：

1. 学习关系型数据库理论：包括关系模型、关系代数、数据库范式、事务理论等，能够帮助你更好地理解数据库的结构和设计。
2. SQL 语言：作为关系型数据库的标准查询语言，掌握 SQL 是非常重要的！学习过程以实践为主，要能够熟练使用 SQL 来操作数据库（增删改查）。关于 SQL 的详细学习，可以查看 [SQL 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755217309741058)
3. 数据库设计和编程：学习如何根据实际业务场景来设计数据库表、并且学习如何使用 C++ 来操作数据库，这是企业开发中最重要的能力。
4. 数据库高级：初学数据库时这部分可以跳过。主要是学习数据库的高级特性、底层原理、数据库的分析、运维、性能优化等，可以帮助你更好地运用数据库来解决问题。

## 阶段 6：C++ 项目实战

这个阶段是最关键的，能不能找到工作，很大程度上取决于你的项目经历。



### 学习建议

1）C++ 项目的特点：由于 C++ 是个相对底层、注重基础的编程语言，很多 C++ 的练手项目都是"手写轮子"相关的，比如手写网络库、手写 Web 服务器、手写 RPC 框架等。C++ 项目对基础要求更高，但也更能锻炼你的编程能力。

2）多看开源项目源码：C++ 的视频项目教程相对较少，更多的是通过阅读开源项目的源码来学习。建议先把项目 clone 下来跑起来，然后阅读源码、理解实现原理。

3）结合求职方向选择项目：简历上的项目最好和你应聘的岗位相关。

4）关于 AI 应用开发：虽然 C++ 主要用于底层和高性能开发，但在 AI 时代，C++ 开发者也可以尝试将 AI 能力集成到项目中。比如在服务器项目中接入大模型 API、开发 AI 推理引擎的 C++ 接口等。现在很多 AI 框架的底层都是用 C++ 实现的（如 TensorFlow、PyTorch 的核心部分），掌握 C++ 和 AI 应用开发的结合，会让你更有竞争力。如果想快速入门 AI 应用开发，可以先通过 Java 或 Python 学习 AI 开发框架，比如鱼皮的 [AI 编程助手](https://www.codefather.cn/course/1943267371799080961)、[AI 零代码应用生成平台](https://www.codefather.cn/course/1948291549923344386) 等项目，理解 AI 应用的开发模式后，再用 C++ 实现高性能版本。

## 最后

C++ 的学习之路确实不容易，相比其他高级语言来说，C++ 的学习曲线更陡峭、需要掌握的底层知识更多。但正是因为难，所以才更有价值！

掌握了 C++，你就掌握了追求极致性能的能力，就能在游戏开发、高性能服务器、音视频处理、嵌入式系统等高技术含量的领域大展拳脚。而且在 AI 时代，懂 C++ 的程序员会更有竞争力，因为很多 AI 框架的底层都是用 C++ 实现的。

学习编程没有捷径，唯有坚持和实践。希望这份学习路线能够帮助大家少走弯路，更高效地掌握 C++。加油小伙伴们！💪
