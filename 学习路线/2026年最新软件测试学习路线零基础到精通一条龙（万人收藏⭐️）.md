## 开篇介绍

软件测试是保证软件质量的重要环节，通过测试可以发现软件中的缺陷（Bug），确保软件符合需求、稳定可靠。

你可能会好奇：既然有开发工程师编写代码，为什么还需要测试工程师呢？

答案很简单：人无完人，代码也一样。即使是经验丰富的开发工程师，也难免会在代码中留下 Bug。测试工程师就像是软件的 “质检员”，在产品发布前发现并修复这些问题，避免让用户遇到糟糕的体验。

随着软件行业的快速发展，软件的复杂度越来越高，功能越来越多，测试的重要性也越来越被重视。现在几乎每个软件团队都配备了专门的测试工程师，甚至有专门的测试团队。测试工程师已经从过去的 “打杂” 角色，发展成为软件开发团队中不可或缺的专业角色。



### 为什么要学软件测试？

软件测试是一个 **入门相对友好** 的技术方向。相比开发岗位，测试岗位对编程能力的要求相对较低，尤其是功能测试，基本不需要编程基础，适合零基础转行。当然，如果想要往自动化测试、性能测试、测试开发方向发展，还是需要掌握编程技能的，但相比纯开发岗位，学习曲线要平缓很多。

市场需求方面，只要有软件开发，就需要软件测试。每个软件产品在发布前都需要经过测试，因此测试岗位的需求量一直很大，而且相对稳定。无论是互联网公司、软件公司，还是传统企业的 IT 部门，都需要测试工程师。

软件测试的职业发展路径也很清晰。你可以从功能测试入门，逐步学习自动化测试、性能测试，最终成为测试开发工程师或测试架构师。每一步的成长都有明确的方向和目标，不会像有些岗位那样感到迷茫。

薪资待遇方面，虽然功能测试的起薪可能不如开发岗位高，但自动化测试和性能测试工程师的薪资水平是很不错的，在一线城市可以达到 10-40K，经验丰富的测试架构师甚至可以达到 20-60K。

![](https://pic.yupi.icu/1/image-20251128114456286.png)

还有一个很多人不知道的优势：测试工程师转开发相对容易。因为测试工程师天天和业务打交道，对需求理解得很深入，如果掌握了编程技能，转开发是非常有优势的。很多优秀的开发工程师都是从测试岗位转过去的。

最后，在 AI 时代，AI 辅助测试成为新趋势。AI 可以帮助生成测试用例、自动化测试脚本编写、智能缺陷分类等，掌握 AI 测试工具将成为测试工程师的加分项。



### 就业方向

学完软件测试后，可以从事以下岗位：

1. 功能测试工程师：执行功能测试，验证软件是否符合需求
2. 自动化测试工程师：编写自动化测试脚本，提高测试效率。关于自动化测试的详细学习，可以查看 [自动化测试学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755686727856129)
3. 性能测试工程师：进行性能测试和压力测试，保证系统性能。关于性能测试的详细学习，可以查看 [性能测试学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755713848225793)
4. 测试开发工程师：开发测试工具和测试平台。关于测试开发的详细学习，可以查看 [测试开发学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755654066810882)
5. 测试架构师 / 测试经理：负责测试团队管理和测试体系建设



### 学习前提

1. 计算机基础：了解基本的计算机概念（不是必须，但有帮助）
2. 编程基础：学习自动化测试需要掌握至少一门编程语言（Python 推荐）
3. 学习时间：功能测试 1-2 个月即可入门，自动化测试需要 3-6 个月

## 整体学习建议

1）理论和实践结合：测试理论知识（如测试用例设计方法、黑盒测试、白盒测试等）是基础，但更重要的是实践。建议边学边练，多做实际项目。

2）优先学习 Python：Python 是测试领域最流行的编程语言，简单易学，拥有丰富的测试框架（Pytest、Unittest）和工具库。建议优先学习 Python。

3）关注现代测试工具：2025 年，Selenium 虽然仍在使用，但已经过时。建议学习现代测试工具，如 Playwright（Web 测试首选）、Cypress（前端测试首选）、Appium（移动端测试标准）。

4）善用 AI 辅助测试：AI 测试是 2025 年的新趋势，可以使用 AI 工具（如 ChatGPT、DeepSeek）生成测试用例、编写测试脚本、分析测试结果。推荐使用 [鱼皮的 AI 资源大全](https://ai.codefather.cn) 中的工具。

![AI资源大全网站](https://pic.yupi.icu/1/AI%E8%B5%84%E6%BA%90%E5%A4%A7%E5%85%A8%E7%BD%91%E7%AB%99.png)

5）注重测试思维：测试不只是执行脚本，更重要的是测试思维。要学会从用户角度思考，发现软件中的潜在问题。

6）多看真实项目：测试离不开业务场景，建议多看真实项目的测试案例，了解不同类型项目（Web、App、接口）的测试方法。

7）持续学习：测试技术发展很快，尤其是 AI 测试、无代码测试等新趋势。建议关注测试技术社区，持续学习新技术。



## 学习路线

### 阶段 1：测试基础理论

了解软件测试的基本概念和理论知识。



#### 知识点

**测试基础：**

- 【必学】什么是软件测试？测试的目的和意义
- 【必学】软件生命周期和测试的关系
- 【必学】测试分类：黑盒测试、白盒测试、灰盒测试
- 【必学】测试级别：单元测试、集成测试、系统测试、验收测试
- 【必学】测试类型：功能测试、性能测试、安全测试、兼容性测试等

**测试用例设计：**

- 【必学】什么是测试用例？测试用例的要素
- 【必学】等价类划分法
- 【必学】边界值分析法
- 【必学】判定表法
- 【建议学】因果图法、正交试验法、场景法

**缺陷管理：**

- 【必学】什么是软件缺陷（Bug）？
- 【必学】缺陷的生命周期
- 【必学】缺陷报告的编写
- 【必学】缺陷管理工具：Jira、禅道、Bugzilla

**测试流程：**

- 【必学】测试计划的制定
- 【必学】测试方案的设计
- 【必学】测试执行和监控
- 【必学】测试报告的编写



#### 学习建议

1）测试基础理论是软件测试的基础，虽然理论性较强，但非常重要。建议先系统学习理论，再进行实践。

2）测试用例设计方法是测试的核心技能，尤其是等价类划分和边界值分析，几乎所有测试都会用到。建议重点学习这两种方法，多做练习。

3）缺陷管理工具（Jira、禅道）是测试工程师的日常工作工具，一定要熟练掌握。Jira 是国际主流工具，禅道是国内主流工具。

![](https://pic.yupi.icu/1/planandtrack.png)

4）测试理论可能比较枯燥，建议结合实际案例学习。比如测试一个登录功能，如何设计测试用例？如何发现和报告缺陷？

### 阶段 2：功能测试实战

学习功能测试的实战技能，这是测试工程师的基本功。



#### 知识点

**Web 功能测试：**

- 【必学】浏览器开发者工具：Elements、Console、Network
- 【必学】Web 测试要点：界面测试、功能测试、兼容性测试
- 【必学】常见 Web 功能测试场景：登录、注册、搜索、表单提交
- 【建议学】浏览器兼容性测试：Chrome、Firefox、Safari、Edge

**App 功能测试：**

- 【必学】App 测试要点：安装卸载、功能测试、兼容性测试
- 【必学】Android 和 iOS 测试差异
- 【建议学】App 性能测试：启动时间、内存占用、电量消耗

**数据库测试：**

- 【必学】SQL 基础：SELECT、INSERT、UPDATE、DELETE
- 【必学】数据库测试要点：数据完整性、数据一致性
- 【建议学】数据库性能测试

**抓包工具：**

- 【必学】Charles / Fiddler：HTTP 抓包工具
- 【必学】Wireshark：网络协议分析工具
- 【建议学】使用抓包工具进行接口测试和调试



#### 学习建议

1）功能测试是测试工程师的基本功，也是入门最快的测试类型。建议多做实际项目，积累测试经验。

2）浏览器开发者工具非常重要，尤其是 Network 标签页，可以查看请求和响应，帮助定位问题。建议熟练掌握 Chrome DevTools。

3）抓包工具（Charles、Fiddler）是测试工程师的必备工具，可以拦截和修改 HTTP 请求，进行接口测试和调试。建议重点学习。

### 阶段 3：接口测试

学习接口测试，这是测试的重要组成部分。



#### 知识点

**接口测试基础：**

- 【必学】什么是接口测试？接口测试 vs 功能测试
- 【必学】HTTP 协议：GET、POST、PUT、DELETE
- 【必学】RESTful API 设计规范
- 【必学】JSON 和 XML 数据格式

**接口测试工具：**

- 【必学】Postman：最流行的接口测试工具
- 【必学】Apifox：国产接口测试工具，功能更强大
- 【建议学】JMeter：性能测试工具，也可用于接口测试
- 【建议学】cURL：命令行接口测试工具

**接口自动化测试：**

- 【必学】使用 Python + Requests 进行接口自动化
- 【建议学】使用 Pytest 编写接口测试用例
- 【建议学】接口测试框架搭建

**接口测试要点：**

- 【必学】接口功能测试：参数校验、返回值校验
- 【必学】接口性能测试：响应时间、并发测试
- 【建议学】接口安全测试：SQL 注入、XSS 攻击



#### 学习建议

1）接口测试是测试的重要组成部分，相比 UI 测试，接口测试更稳定、更快速。建议重点学习。

2）Postman 和 Apifox 是接口测试的必备工具。Postman 是国际主流工具，Apifox 是国产工具，功能更强大（支持接口文档、Mock、自动化测试等）。建议两个都学习。

![](https://pic.yupi.icu/1/rest-client-send-and-save-postman.png)

3）接口自动化测试建议使用 Python + Requests + Pytest，这是最流行的技术栈，简单易学。

4）接口测试的核心是理解 HTTP 协议和 RESTful API 设计规范。建议系统学习 HTTP 协议，了解请求和响应的结构。

5）多做实际练习：找一些开放的 API（如 GitHub API、天气 API）进行测试练习，熟悉接口测试的流程。

### 阶段 4：自动化测试（重点）

学习自动化测试，这是测试工程师进阶的关键。



#### 知识点

**自动化测试基础：**

- 【必学】什么是自动化测试？自动化测试的优势和局限
- 【必学】自动化测试的适用场景
- 【必学】自动化测试框架的选择

**编程语言：**

- 【必学】Python：测试领域最流行的语言，简单易学
- 【建议学】JavaScript / TypeScript：前端测试的首选语言

**Web 自动化测试（重点）：**

- 【必学】Playwright（推荐）：微软出品的现代化 Web 测试框架
  - 支持多浏览器（Chrome、Firefox、Safari、Edge）
  - 自动等待机制，测试更稳定
  - 支持录制和生成测试脚本
  - 性能优秀，执行速度快
- 【必学】Cypress（推荐）：前端开发者最喜欢的测试框架
  - 实时重载，开发体验好
  - 强大的调试功能
  - 适合前端开发者使用
- 【建议学】Selenium（传统工具）：虽然是传统工具，但仍有公司在使用
  - 学习建议：简单了解即可，重点学习 Playwright 和 Cypress

**移动端自动化测试：**

- 【必学】Appium：移动端自动化测试的标准工具
  - 支持 Android 和 iOS
  - 基于 WebDriver 协议
- 【建议学】Appium Inspector：Appium 的元素定位工具

**测试框架：**

- 【必学】Pytest：Python 测试框架，功能强大
- 【必学】Jest / Vitest：JavaScript 测试框架
- 【建议学】TestNG：Java 测试框架（如果使用 Java）

**Page Object Model（POM）：**

- 【必学】什么是 POM？POM 的优势
- 【必学】POM 的实现和应用
- 【必学】POM + 数据驱动测试

**持续集成（CI/CD）：**

- 【建议学】Jenkins：传统 CI/CD 工具
- 【建议学】GitHub Actions：最简单的 CI/CD 工具
- 【建议学】GitLab CI：GitLab 自带的 CI/CD 工具



#### 学习建议

1）自动化测试是测试工程师进阶的关键，掌握自动化测试可以大大提升测试效率和薪资水平。

2）Web 自动化测试工具的选择：
   - Playwright：2025 年 Web 自动化测试的首选，功能强大、性能优秀、API 简洁。建议重点学习。
   - Cypress：前端开发者最喜欢的测试框架，开发体验好。如果你熟悉前端开发，建议学习 Cypress。
   - Selenium：虽然是传统工具，但仍有公司在使用。建议简单了解，但不要花太多时间深入学习。

3）Playwright 的优势：
   - 自动等待机制：不需要手动添加 `time.sleep()`，测试更稳定
   - 录制功能：可以录制浏览器操作，自动生成测试脚本
   - 多浏览器支持：一套代码可以在多个浏览器运行
   - 性能优秀：并行执行测试，速度快

4）POM（Page Object Model）是自动化测试的最佳实践，可以提高代码的可维护性和可读性。建议在自动化测试项目中应用 POM。

5）持续集成（CI/CD）可以实现自动化测试的自动运行，是自动化测试的重要组成部分。建议学习 GitHub Actions，它是最简单的 CI/CD 工具。

6）善用 AI 编写测试脚本：可以使用 AI 工具（如 ChatGPT、GitHub Copilot）帮助编写测试脚本，大大提高开发效率。

### 阶段 5：性能测试

学习性能测试，保证系统在高并发下的稳定性。

#### 知识点

**性能测试基础：**

- 【必学】什么是性能测试？性能测试的目的
- 【必学】性能测试类型：负载测试、压力测试、稳定性测试
- 【必学】性能指标：响应时间、吞吐量、并发数、TPS、QPS
- 【建议学】性能测试流程：需求分析、场景设计、执行测试、结果分析

**性能测试工具：**

- 【必学】JMeter：最流行的性能测试工具
  - 开源免费
  - 支持多种协议（HTTP、HTTPS、FTP、JDBC 等）
  - 可视化界面，易于使用
- 【建议学】K6：现代化的性能测试工具
  - 使用 JavaScript 编写测试脚本
  - 性能优秀，支持大规模测试
  - 云原生，易于集成 CI/CD
- 【建议学】Gatling：基于 Scala 的性能测试工具
  - 性能优秀，支持大规模测试
  - 测试报告美观
- 【可不学】LoadRunner：商业工具，功能强大但价格昂贵

**JMeter 核心功能：**

- 【必学】线程组：模拟并发用户
- 【必学】HTTP 请求：发送 HTTP 请求
- 【必学】断言：验证响应结果
- 【必学】监听器：查看测试结果
- 【建议学】参数化：使用 CSV 数据文件
- 【建议学】关联：提取响应数据用于后续请求

**性能测试实战：**

- 【必学】Web 应用性能测试
- 【必学】接口性能测试
- 【建议学】数据库性能测试
- 【建议学】性能瓶颈分析和调优



#### 学习建议

1）JMeter 是性能测试的首选工具，开源免费、功能强大、社区活跃。建议重点学习 JMeter。

![](https://pic.yupi.icu/1/d365bcapiloadtestjmeter_15.jpg)

2）K6 是 2025 年新兴的性能测试工具，使用 JavaScript 编写测试脚本，性能优秀。如果你熟悉 JavaScript，可以学习 K6。

3）性能测试的核心是场景设计和结果分析。建议学习如何设计合理的测试场景（如逐步增加并发用户），如何分析测试结果（如响应时间分布、吞吐量变化）。

4）性能测试需要一定的服务器和网络知识，建议补充学习 Linux 服务器、网络协议等知识。

5）强烈推荐通过鱼皮的实战项目学习 JMeter。鱼皮的 [聚合搜索平台](https://www.codefather.cn/course/1790979621621641217) 和 [代码生成器共享平台](https://www.codefather.cn/course/1790980795074654209) 都从 0 到 1 带你入门并实战 JMeter 压力测试，包括线程组配置、HTTP 请求设置、结果分析等，比单纯学习理论效果好得多。
