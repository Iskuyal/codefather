## 开篇介绍

自动化测试是软件测试领域最重要的技能方向之一，通过编写脚本和使用工具，让原本需要手工执行的测试工作实现自动化。随着敏捷开发和 DevOps 的普及，自动化测试的重要性越来越高。

**为什么要学自动化测试？**

1）自动化测试是测试工程师的核心技能。根据今年的市场调研，掌握自动化测试技能的测试工程师薪资比只会手工测试的高 30-50%。

2）自动化测试能够提升工作效率。通过自动化，可以帮你从繁重的重复工作中解放出来，专注于更有价值的测试工作。它可以大幅提高测试效率、降低人力成本、提升测试覆盖率、保证回归测试的稳定性。

目前，自动化测试是测试开发工程师的必备技能，广泛应用于各种场景：回归测试、冒烟测试、兼容性测试、性能测试、持续集成等。再加上如今 AI 的爆炸式发展，也进一步提高了自动化测试的上限。



### 就业方向

学习自动化测试可以从事多个方向的工作：

1. 自动化测试工程师：负责自动化测试脚本的开发和维护，执行自动化测试，分析测试结果。
2. 测试开发工程师：不仅会测试，还能开发测试工具和测试平台，是测试和开发的结合。关于测试开发的详细学习，可以查看 [测试开发学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755654066810882)
3. 性能测试工程师：专注于性能测试，使用自动化工具进行压力测试、负载测试等。关于性能测试的详细学习，可以查看 [性能测试学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755713848225793)
4. 质量保障工程师（QA）：负责整体的质量保障工作，包括测试策略制定、自动化测试推广等。
5. DevOps 工程师：将自动化测试集成到 CI/CD 流程中，实现持续测试和持续交付。

## 阶段 1：编程基础（15-30 天，仅供参考）

Python 是自动化测试最常用的编程语言，简单易学，库丰富。如果你完全零基础，建议系统地学习 Python，能够独立编写脚本。如果你已经会其他编程语言，可以快速过一遍 Python 语法。

关于 Python 的详细学习，可以查看 [Python 学习路线](https://www.codefather.cn/course/1789189862986850306/section/1789190283176419330)，系统地掌握 Python 编程。



### 学习目标

掌握 Python 编程基础，为编写自动化测试脚本打下基础。



### 知识点

**Python 基础【必学】：**

- Python 语法
- 数据类型和操作
- 控制流程
- 函数和模块
- 面向对象编程
- 异常处理
- 文件操作

**常用库【必学】：**

- re（正则表达式）
- json（JSON 处理）
- requests（HTTP 请求）
- pytest（测试框架）

**版本控制【必学】：**

- Git 基础
- GitHub/GitLab 使用



### 学习建议

1）自动化测试不需要掌握 Python 的全部特性，重点掌握基础语法、面向对象、常用库即可。高级特性（如装饰器、元类）可以在需要时再学习。

2）pytest 是 Python 最流行的测试框架，必须掌握。pytest 的特点是简单灵活，支持参数化、fixture、插件等功能。后续的自动化测试都会基于 pytest。

## 阶段 2：Web 自动化测试（20-40 天，仅供参考）


通过 Selenium、Playwright 等工具模拟用户操作，实现 Web 应用的自动化测试。



### 学习目标

掌握 Web 自动化测试技术，能够使用 Selenium 等工具编写 Web 测试脚本。



### 知识点

**Selenium 基础【必学】：**

- Selenium 架构
- WebDriver 安装和配置
- 浏览器驱动
- 元素定位（ID、Name、XPath、CSS Selector）
- 常用操作（点击、输入、下拉框、弹窗）

**Selenium 高级特性【必学】：**

- 等待机制（显式等待、隐式等待）
- 多窗口处理
- Frame 切换
- JavaScript 执行
- 截图和日志

**自动化测试框架【必学】：**

- POM（Page Object Model）
- 数据驱动测试
- 关键字驱动测试
- 测试用例管理

**其他 Web 自动化工具【建议学】：**

- Playwright（新一代 Web 自动化工具）
- Cypress（前端开发者友好）
- 工具对比和选型



### 学习建议

1）Selenium 是 Web 自动化测试的行业标准，必须熟练掌握。Selenium 4 带来了很多新特性，建议学习最新版本。元素定位是 Selenium 的核心，要掌握多种定位方法，优先使用稳定的定位方式（如 ID、Name），XPath 和 CSS Selector 作为备选。

![](https://pic.yupi.icu/1/image-20251203123631814.png)

2）页面加载需要时间，如果不等待就操作元素，会导致测试失败。要理解显式等待和隐式等待的区别，合理使用等待。推荐使用显式等待（WebDriverWait），更加灵活可控。

3）POM（页面对象模型）是组织 Selenium 代码的最佳实践。将页面元素和操作封装到 Page 类中，测试用例只调用 Page 类的方法。这样可以提高代码的可维护性和复用性。

4）Playwright 是新一代 Web 自动化工具，比 Selenium 更快、更稳定，支持多浏览器、自动等待、录制功能等。**如果是新项目，建议优先考虑 Playwright。**

5）建议从简单的页面开始练习，比如登录页面、搜索页面等。逐步增加难度，处理复杂的交互、多窗口、Frame 等场景。在实践中积累经验。

## 阶段 3：移动端自动化测试（15-30 天，仅供参考）

### 学习目标

掌握移动端自动化测试技术，能够使用 Appium 编写移动应用测试脚本。



### 知识点

**Appium 基础【必学】：**

- Appium 架构
- 环境搭建（Android SDK、Xcode）
- Desired Capabilities 配置
- 元素定位（ID、XPath、Accessibility ID）
- 常用操作（点击、滑动、输入）

**Android 自动化【必学】：**

- Android 应用结构
- uiautomatorviewer
- 元素定位技巧
- 手势操作

**iOS 自动化【建议学】：**

- iOS 应用结构
- Xcode Instruments
- 元素定位技巧

**Appium 高级特性【必学】：**

- 多设备并行测试
- 混合应用（Hybrid App）测试
- WebView 测试
- 真机测试 vs 模拟器测试



### 学习建议

1）Appium 是移动端自动化测试的主流工具，支持 Android 和 iOS。Appium 基于 WebDriver 协议，和 Selenium 类似，学过 Selenium 可以快速上手 Appium。

![](https://pic.yupi.icu/1/Appium-OverView.png)

2）环境搭建是 Appium 的第一个难点。Android 需要安装 JDK、Android SDK、配置环境变量；iOS 需要 Mac 电脑和 Xcode，足以劝退不少同学了。

建议先在 Android 上学习，环境搭建相对简单。

3）移动端的元素定位比 Web 更复杂。要学会使用 uiautomatorviewer（Android）或 Appium Inspector 查看元素属性，选择合适的定位方式。优先使用 resource-id，XPath 作为备选。

4）手势操作是移动端特有的交互方式。要掌握滑动、长按、拖拽等手势操作的实现方法。Appium 提供了 TouchAction 类用于实现各种手势。

5）Appium 2.0 是最新版本，架构有较大变化，建议学习 Appium 2.0。要理解 Driver 和 Plugin 的概念，掌握新的配置方式。

## 阶段 4：接口自动化测试（15-30 天，仅供参考）

接口测试是自动化测试中最重要的一部分，投入产出比最高。接口相对稳定，自动化后维护成本低，执行速度快。建议重点学习接口自动化测试。



### 学习目标

掌握接口自动化测试技术，能够使用 requests、pytest 等工具编写接口测试脚本。



### 知识点

**接口测试基础【必学】：**

- HTTP 协议
- RESTful API
- 接口文档阅读
- 接口测试方法

**requests 库【必学】：**

- GET、POST 请求
- 请求头和请求体
- 响应处理
- Session 管理
- 文件上传下载

**pytest 进阶【必学】：**

- 参数化测试
- Fixture
- 断言
- 测试报告
- 插件使用

**接口自动化框架【必学】：**

- 框架设计
- 数据驱动
- 关键字驱动
- 配置管理
- 日志管理



### 学习建议

1）requests 是 Python 最流行的 HTTP 库，API 简洁易用。要掌握各种请求方法、参数传递、响应处理等。结合 pytest，可以快速搭建接口自动化测试框架。

2）参数化测试是 pytest 的强大功能，可以用相同的测试逻辑测试多组数据。要掌握 @pytest.mark.parametrize 的使用，实现数据驱动测试。

3）建议学习完整的 [接口测试学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755744693137410)，深入掌握接口测试技术。



### 学习资源

- [接口测试学习路线](https://www.codefather.cn/course/1789189862986850306/section/1990755744693137410)：完整的接口测试路线

## 阶段 5：自动化测试框架（20-40 天，仅供参考）

### 学习目标

掌握自动化测试框架的设计和实现，能够搭建企业级自动化测试框架。



### 知识点

**框架设计【必学】：**

- 分层架构（测试层、业务层、驱动层）
- 目录结构设计
- 配置管理
- 数据管理
- 日志管理

**POM 设计模式【必学】：**

- Page Object Model
- 页面类设计
- 元素封装
- 方法封装

**数据驱动【必学】：**

- Excel 数据驱动
- CSV 数据驱动
- YAML/JSON 配置
- 参数化测试

**测试报告【必学】：**

- Allure 报告
- HTML 报告
- 测试结果统计
- 失败截图

**CI/CD 集成【必学】：**

- Jenkins 集成
- GitLab CI 集成
- GitHub Actions 集成
- 定时执行



### 学习重点

1）自动化测试框架是自动化测试的核心。好的框架可以提高开发效率、降低维护成本、提升测试稳定性。要学习如何设计一个易用、稳定、可扩展的自动化测试框架。

2）分层架构是框架设计的重要原则。测试层负责测试用例，业务层负责业务逻辑封装，驱动层负责工具调用。分层后，各层职责清晰，代码更易维护。

3）Allure 是最流行的测试报告框架，报告美观、功能强大。要掌握 Allure 的使用，包括添加描述、添加附件、失败重试等功能。Allure 报告可以很好地展示测试结果。

4）CI/CD 集成是自动化测试的重要环节。要学会将自动化测试集成到 Jenkins、GitLab CI 等平台，实现代码提交后自动执行测试。这样可以及时发现问题，保证代码质量。

5）建议参考开源的自动化测试框架，学习别人的设计思路。也可以参加一些自动化测试的技术分享会，了解业界的最佳实践。



### 学习资源

- [Allure 官方文档](https://docs.qameta.io/allure/)：测试报告框架
- [Jenkins 官方文档](https://www.jenkins.io/doc/)：CI/CD 工具



## 阶段 6：项目实战（25-50 天，仅供参考）

### 学习目标

通过完整的项目实践，综合运用所学知识，积累自动化测试经验。



### 学习建议

1）搭建一个完整的自动化测试项目。可以选择一个开源的 Web 应用或移动应用作为测试对象，编写完整的自动化测试用例，覆盖主要功能。

2）项目要包含完整的框架：分层架构、POM 设计、数据驱动、测试报告、CI/CD 集成。不要只写测试用例，要设计好框架结构，提高代码质量。

3）关注测试覆盖率和稳定性。测试覆盖率要达到 80% 以上，测试用例的成功率要达到 95% 以上。不稳定的测试用例（Flaky Tests）要及时修复。

4）将项目开源到 GitHub，写好 README，提供运行指南。这样可以作为你的作品展示给面试官，大大提升你的竞争力。

5）建议加入自动化测试社区，和其他测试工程师交流。可以在技术论坛、微信群、QQ 群等平台提问和分享经验。



### 项目推荐

- Web 电商平台自动化测试
- 移动应用自动化测试
- 接口自动化测试框架
- 混合型自动化测试平台

**优质开源项目：**

- ⭐ [Playwright](https://github.com/microsoft/playwright)：68k+ stars，Microsoft 现代化 Web 自动化测试框架

## 总结一下

自动化测试是软件测试领域最重要的技能方向，掌握自动化测试能够让你在测试领域获得更好的发展。从 Python 编程到 Selenium Web 测试，从 Appium 移动端测试到接口自动化测试，从框架设计到 CI/CD 集成，自动化测试的技术体系完整且实用。

学习自动化测试要先打好编程基础，掌握 Python 语言和 pytest 框架。深入学习 Web 自动化测试，掌握 Selenium 或 Playwright 的使用。学习移动端自动化测试，掌握 Appium 的使用。掌握接口自动化测试技术。设计和实现完整的自动化测试框架。将自动化测试集成到 CI/CD 流程。通过完整的项目实践，积累开发经验。

记住，自动化测试不是万能的，要根据项目特点选择合适的测试策略。自动化测试的成本包括脚本开发成本和维护成本，要权衡投入产出比。选择合适的测试对象、使用合适的工具、设计好的框架，才能发挥自动化测试的最大价值。
